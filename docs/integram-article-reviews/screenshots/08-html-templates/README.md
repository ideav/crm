# Скриншоты для статьи 08-html-templates

Папка для скриншотов к статье `docs/integram-article-reviews/08-html-templates.md`. Файлы добавляются перед публикацией статьи и 1-минутного видео-обзора.

## Что снять

| Файл | Что должно быть на скриншоте | Где в Интеграме |
| --- | --- | --- |
| `01-templates-folder.png` | Содержимое папки `templates/` с файлами `cards.html`, `dash.html`, `info.html`, `form.html` и другими | Файловая система рядом с экземпляром Интеграма |
| `02-info-html-placeholders.png` | Открытый `templates/info.html` с подсветкой блоков `{ЗапросID}`, `{Запрос}`, `{Описание}` и пары `<!-- Begin:Пользовательские -->` / `<!-- End:Пользовательские -->` | Текстовый редактор или GitHub-просмотр файла |
| `03-rendered-workplace.png` | Готовое рабочее место, собранное из HTML-шаблона: список отчётов и форм, подменённые значения полей | Экран Интеграма после загрузки шаблона |
| `04-dispatcher-dashboard.png` (опционально) | Пример сценария из статьи: дашборд с картой, списком заявок и счётчиками | Экран Интеграма с подключённой картой |
| `05-airtable-or-notion-limits.png` (опционально) | Контрпример: попытка вставить произвольный HTML/JS в Airtable Interfaces или Notion view | Скриншот соответствующего интерфейса |

## Требования

- Скриншоты делаются на реальном экземпляре Интеграма, не на макетах.
- Чувствительные данные (имена клиентов, телефоны, суммы) замаскированы или заменены тестовыми.
- Численные сравнения в подписях (лимиты Airtable Interfaces, Notion API) перепроверяются на день публикации.
- Формат — PNG, ширина 1280–1920 px, без оптимизации цвета (gamma по умолчанию).
