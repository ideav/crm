# Права доступа из коробки: почему Интеграм безопаснее общей таблицы

**Сравнение:** Excel, Google Sheets, простые no-code-таблицы

---

## Контекст

Общий доступ к документу — распространённая модель совместной работы. Ссылку на Google Sheet можно отправить коллеге с правами «Редактор» или «Читатель». В Excel Online аналогично. Для небольших команд с однородными задачами этого достаточно.

Проблема возникает, когда команда неоднородна: одни сотрудники должны видеть свои данные, другие — все данные, третьи — только читать, четвёртые — только вносить записи через форму. Инструменты типа «общей таблицы» для этого не проектировались.

---

## Конкретный сценарий

Отдел продаж: 10 менеджеров, 2 руководителя, 1 администратор CRM, 1 финансист. Требования к доступу:

- менеджер видит только своих клиентов и сделки, не видит данные коллег;
- руководитель видит все данные своего отдела, не видит финансовые поля стоимости сделки;
- финансист видит финансовые поля, не видит переписку;
- администратор управляет структурой, не работает с оперативными данными;
- новый партнёр заполняет форму заявки без доступа к внутренним данным.

В Google Sheets эту схему реализовать нельзя средствами платформы. Обходные варианты — скрытые листы, скрипты Apps Script с проверкой email, отдельные файлы для каждой роли — создают дополнительную сложность и новые точки отказа.

---

## Что делает Интеграм иначе

**Роли.** В Интеграме создаётся роль для каждого типа пользователя. Роли назначают набор доступных таблиц и уровень доступа: чтение, редактирование, создание, удаление.

**Маски по значениям.** Менеджер видит только строки, где поле «Ответственный» равно его имени. Это настраивается на уровне роли без дополнительного программирования.

**Скрытые поля.** Финансовые поля скрыты для ролей, которым они не нужны. Скрытое поле не передаётся в интерфейс — пользователь не может «подсмотреть» его в коде страницы.

**Гостевой доступ.** Публичная форма для внешних заявок не требует учётной записи. Данные попадают в базу, но заявитель не получает доступа к внутренним таблицам.

---

## Ограничения Интеграма в этом сценарии

Гранулярность прав в Интеграме достаточна для типовых бизнес-сценариев, но уступает специализированным системам класса ERP или корпоративным LDAP-решениям. Если требуются сложные иерархические права с наследованием, аудит-лог на уровне отдельных изменений полей и интеграция с корпоративной директорией — нужно оценивать конкретные требования отдельно.

---

## Вывод

Разграничение доступа в Интеграме — встроенная возможность, а не обходное решение. Менеджер видит своих клиентов, руководитель видит всех, финансист видит финансовые поля, внешний партнёр заполняет форму без доступа к базе. Эта модель настраивается без дополнительного программирования.

## Раскадровка 1-минутного видео

**0:00–0:10 — Хук**
Экран: Google Sheets с одним уровнем доступа «Редактор» для всех.
Текст за кадром: «Все менеджеры видят данные друг друга. Это не всегда нужно.»

**0:10–0:35 — Демонстрация**
Экран 1: настройка роли «Менеджер» — маска по значению «Ответственный = текущий пользователь».
Экран 2: вид рабочего места менеджера — только свои записи.
Экран 3: публичная форма заявки без логина.

**0:35–0:55 — Сравнение**
Текст за кадром: «Совместный доступ к документу — это не управление правами. Это разные задачи.»

**0:55–1:00 — Следующий шаг**
Текст на экране: ссылка на документацию по ролям.
