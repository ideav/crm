# Нет потолка одного листа Excel: как Интеграм помогает уйти от лимита 1 048 576 строк

**Сравнение:** Excel

---

## Контекст

Excel — стандартный инструмент для расчётов, сводных таблиц и анализа. Миллионы сотрудников работают с ним ежедневно, и для многих задач он остаётся оптимальным выбором.

Другая ситуация — когда Excel используется не как инструмент анализа, а как единственное хранилище процесса: записи добавляются ежедневно, файл передаётся коллегам, из него строятся зависимые вычисления. Именно здесь возникают характерные проблемы.

---

## Конкретный сценарий

Организация ведёт в Excel реестр обращений или операций. Данные накапливаются несколько лет. Типичные точки роста:

- лист Excel ограничен 1 048 576 строками (технический лимит Microsoft, задокументированный в официальной спецификации). При достижении лимита новые строки физически не добавляются;
- до достижения лимита файл тормозит: расчёт формул на 500 000 строк занимает заметное время;
- данные разнесены по нескольким файлам за разные периоды — ВПР между файлами, ручное объединение для анализа;
- файл нельзя безопасно открыть параллельно двум сотрудникам: изменения одного перезапишут изменения другого при сохранении.

---

## Что делает Интеграм иначе

**Хранение без строкового лимита.** Данные хранятся в реляционной базе, а не в листе с фиксированным числом строк. Ограничение в 1 048 576 строк к Интеграму не применимо.

**Нормализация вместо плоской таблицы.** Плоский Excel-реестр с повторяющимися полями (наименование контрагента в каждой строке) можно перенести в связанные таблицы: контрагенты — отдельно, операции — отдельно, ссылка — через реляционное поле. Изменение названия контрагента отразится везде автоматически.

**Параллельная работа.** Несколько пользователей редактируют данные одновременно без конфликтов сохранения.

**Импорт и экспорт.** Существующий Excel-файл можно импортировать в Интеграм. Для аналитики данные доступны в экспорте обратно в Excel или через JSON API.

---

## Ограничения Интеграма в этом сценарии

Интеграм не заменяет расчётные возможности Excel: формулы между ячейками, сводные таблицы, диаграммы «из коробки» средствами самого Excel здесь не воспроизводятся. Сценарий использования Интеграма — хранение и структурирование данных, а не их анализ в стиле электронной таблицы. Для аналитики данные экспортируются в Excel или подключаются через API к BI-инструменту.

---

## Вывод

Excel остаётся сильным инструментом для расчётов и единовременного анализа. Роль постоянно растущего хранилища оперативных данных он выполняет хуже: ограничение строк, отсутствие параллельной работы и хрупкость формул при больших объёмах создают предсказуемые проблемы. Интеграм берёт на себя роль хранения и структурирования, оставляя Excel для аналитических выгрузок.
