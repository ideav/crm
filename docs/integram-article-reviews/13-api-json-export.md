# API и JSON-экспорт вместо копипаста: как Интеграм связывает учет с внешними системами

**Сравнение:** Excel, Google Sheets, Notion, ручные выгрузки

---

## Контекст

Данные редко живут в изоляции. Реестр договоров нужен финансовой системе. Данные о клиентах — маркетинговой платформе. Статус заявки — внешнему сервису уведомлений. Когда инструмент учёта не поддерживает автоматического обмена данными, интеграция строится на ручных выгрузках: экспортировал Excel, отправил почтой, коллега вручную загрузил в другую систему.

---

## Конкретный сценарий

Компания ведёт учёт в Excel/Google Sheets. Финансовый отдел ежедневно нужны данные для сводного отчёта. Текущий процесс:

1. менеджер экспортирует Excel-файл;
2. отправляет его финансисту;
3. финансист вручную переносит данные в свою таблицу;
4. при ошибке в исходном файле — повторяет шаги 1–3.

Такая же схема для BI-инструмента: аналитик раз в неделю вручную загружает свежий CSV. Данные в BI отстают на неделю.

---

## Что делает Интеграм иначе

**JSON-экспорт отчёта.** Любой отчёт в Интеграме доступен в формате JSON по URL. Внешняя система запрашивает URL — получает актуальные данные без ручного участия.

**Серверный API.** Интеграм предоставляет API для операций с данными: чтение записей, создание, обновление. API работает с теми же правами, что и пользовательский интерфейс.

**API-пользователь.** Для внешней системы создаётся отдельный пользователь с ограниченными правами: только чтение нужных таблиц. Административный доступ не передаётся.

**Подключение BI.** Power BI, Metabase и другие BI-инструменты умеют получать данные через JSON-эндпоинты или прямое SQL-подключение (зависит от конфигурации). Данные в BI обновляются по расписанию автоматически.

---

## Ограничения Интеграма в этом сценарии

API Интеграма — это API конкретной платформы, а не универсальный стандарт вроде REST с OpenAPI-спецификацией. Интеграция требует изучения структуры запросов. Для сложных сценариев (двусторонняя синхронизация, событийные вебхуки в реальном времени) возможности зависят от конкретной конфигурации.

---

## Вывод

Ручная выгрузка данных — это операционная нагрузка и источник ошибок. JSON-API Интеграма позволяет внешним системам забирать данные напрямую, без участия сотрудника. Данные остаются актуальными, а финансовый или BI-инструмент не зависит от того, кто и когда отправил файл.
