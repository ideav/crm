# Сценарий ввода данных Atex по ролям для #3027

**Задача:** [#3027](https://github.com/ideav/crm/issues/3027)
**Цель:** проверить полный производственный путь через ввод данных тестовыми
пользователями и нажатие кнопок рабочих мест Atex: каждый пользователь вводит
информацию и нажимает кнопку своего шага, а не только смотрит статусы.

Сценарий использует метку `АТХ-3027`. Для каждого запуска добавляйте суффикс
даты или номера, например `АТХ-3027-2026-05-31-01`, чтобы созданные записи было
легко найти в списках и дашбордах.

## Участники

| Роль | Пользователь | Рабочее место |
|---|---|---|
| Менеджер | `manager` | `/atex/orders` |
| Диспетчер | `dispatcher` | `/atex/cut-calc`, `/atex/production-planning` |
| Оператор | `operator` | `/atex/sleeve-cutter` |
| Руководитель | `director` | `/atex/dashboards` |

## Контрольные данные

| Поле | Значение |
|---|---|
| Клиент | `ООО Ромашка-Термолента #3027` или существующий тестовый клиент |
| Вид сырья | `MWR118` или существующий тестовый вид сырья |
| Партия сырья | существующая партия с остатком не меньше `1200` |
| Слиттер | любой доступный тестовый слиттер |
| Втулкорез | любой доступный тестовый втулкорез |
| Нетиповая резка | `TT-АТХ-3027 57x10+40x8` |
| Ширина входа, мм | `910` |
| Допуск, мм | `25` |
| Полоса 1 | ширина `57`, количество `10`, назначение `Заказ` |
| Полоса 2 | ширина `40`, количество `8`, назначение `Заказ` |
| Расчёт | `Итого ножей = 18`, `Остаток, мм = 20` |
| Позиция заказа | количество `10`, ширина `57`, длина `1200`, диаметр втулки `76` |
| Обеспечение | метраж `1200`, статус `Зарезервировано` |
| Задание на втулки | план `10`, факт `10`, диаметр `76` |

## Менеджер

Менеджер вводит заказ и позицию в рабочем месте `/atex/orders`.

1. Открыть `/atex/orders` под пользователем `manager`.
2. Нажать `Создать заказ`.
3. Выбрать клиента.
4. Оставить статус `Новый`.
5. Ввести в `Примечания`: `АТХ-3027: клиент просит: нетиповая резка 57x10+40x8`.
6. Нажать `Создать заказ`.
7. Раскрыть созданный заказ.
8. Нажать `Добавить позицию`.
9. Выбрать вид сырья и рассчитанный тип резки `TT-АТХ-3027 57x10+40x8`.
10. Ввести `Кол-во = 10`, `Ширина, мм = 57`, `Длина, м = 1200`,
    `Ø втулки = 76`.
11. Нажать `Сохранить позицию`.
12. Перевести статус заказа в `Согласован`, статус позиции в `В работе`.

Запись выполняется через `_m_new` для `Заказ` и `Позиция заказа`, смена статусов
выполняется через `_m_set`.

## Диспетчер

Диспетчер вводит тип резки для случая `нетиповая резка`, сохраняет расчёт и
планирует производство.

1. Открыть `/atex/cut-calc` под пользователем `dispatcher`.
2. Нажать `Новый`, если форма уже открыта на существующей записи.
3. Ввести `Название = TT-АТХ-3027 57x10+40x8`.
4. Выбрать вид сырья.
5. Ввести `Ширина входа, мм = 910`, `Допуск, мм = 25`.
6. В первой строке полос ввести `57` и `10`, назначение `Заказ`.
7. Нажать `+ Добавить полосу`.
8. Во второй строке полос ввести `40` и `8`, назначение `Заказ`.
9. Проверить расчёт `Итого ножей = 18` и `Остаток, мм = 20`.
10. Нажать `Сохранить`.
11. Открыть `/atex/production-planning`.
12. Выбрать слиттер, тип резки `TT-АТХ-3027 57x10+40x8` и партию сырья.
13. Ввести дату плана и статус `В очереди`.
14. Нажать `Создать резку`.
15. Выбрать созданную резку в очереди.
16. В блоке обеспечения выбрать позицию заказа менеджера.
17. Ввести `Метраж, м = 1200`, оставить статус `Зарезервировано`.
18. Нажать `Привязать к резке`.

Расчёт типа резки записывает родительскую запись `Тип резки` и дочерние записи
`Полоса` через `_m_new`. Планирование записывает `Производственная резка` через
`_m_new`, затем связывает позицию заказа через запись `Обеспечение`.

## Оператор

Оператор вводит задание на втулки и переводит его кнопками по цепочке статусов.

1. Открыть `/atex/sleeve-cutter` под пользователем `operator`.
2. Выбрать созданную производственную резку `TT-АТХ-3027 57x10+40x8`.
3. Нажать `+ Добавить задание`.
4. Выбрать втулкорез.
5. Ввести `Диаметр, мм = 76`, `Кол-во план = 10`, `Кол-во факт = 10`.
6. Нажать `Сохранить`.
7. Нажать `→ В работе`.
8. Нажать `→ Готово`.

Задание на втулки создаётся через `_m_new`, дальнейшие изменения полей и статуса
выполняются через `_m_set`.

## Руководитель

Руководитель открывает `/atex/dashboards` под пользователем `director` после
ввода данных всеми ролями и проверяет, что строка `АТХ-3027` видна в
`Путь продукции`: заказ, позиция, обеспечение, резка и ГП/отгрузка отображаются
как этапы одного производственного пути.

## Автоматическая регрессия

Минимальная проверка сценария:

```bash
node experiments/test-issue-3027-atex-role-input-walkthrough.js
```

Тест закрепляет, что сценарий содержит ввод значений и кнопки рабочих мест, а
чистые помощники рабочих мест формируют записи `_m_new` и `_m_set` для заказа,
позиции, расчёта нетиповой резки, планирования и задания на втулки.
