# Кастомные настройки диаграммы (`customOptions`)

В разделе **Варианты отображения панели → Диаграммы** для каждого типа диаграммы доступно поле **«Кастомные настройки диаграммы (JSON)»**. Оно позволяет переопределить настройки диаграммы Chart.js, не меняя код приложения.

Поле принимает один JSON-объект с тремя необязательными ключами:

| Ключ | Назначение |
| --- | --- |
| `options` | Объект, который будет глубоко смёржен поверх итоговых `options` диаграммы Chart.js (после применения общих настроек). |
| `dataset` | Переопределения, применяемые к **каждому** dataset диаграммы. |
| `datasets` | Переопределения для конкретных dataset'ов: либо массив (по индексу), либо объект, где ключ — индекс. Поддерживаются отрицательные индексы (`-1` — последний dataset). |

Внутри `dataset` и каждого элемента `datasets` поддерживаются:

- любые поля Chart.js dataset (например, `backgroundColor`, `borderColor`, `borderWidth`, `tension`);
- специальные ключи `pointColors` и `pointBorderColors` — карта `индекс → цвет` для отдельных точек / столбиков. Ключи могут быть положительными (`0`, `1`, …) или отрицательными (`-1` — последний элемент).

Если JSON некорректный или нарушает структуру — настройки игнорируются, а под полем выводится сообщение об ошибке.

## Пример: последний столбик другого цвета

Чтобы выделить последний столбик столбчатой диаграммы красным, задайте:

```json
{
  "datasets": {
    "-1": {
      "pointColors": {
        "-1": "#e53935"
      }
    }
  }
}
```

Если все значения находятся в первом dataset (типичный случай для одной серии), используйте `dataset`:

```json
{
  "dataset": {
    "pointColors": {
      "-1": "#e53935"
    }
  }
}
```

## Пример: толщина линии и пунктир

```json
{
  "dataset": {
    "borderWidth": 3,
    "borderDash": [6, 4]
  }
}
```

## Пример: переопределение опций Chart.js

Скрыть легенду и подписать ось Y:

```json
{
  "options": {
    "plugins": {
      "legend": { "display": false }
    },
    "scales": {
      "y": {
        "title": { "display": true, "text": "Сумма, ₽" }
      }
    }
  }
}
```

## Пример: индивидуальный цвет каждой точки

```json
{
  "dataset": {
    "pointColors": {
      "0": "#1B50F3",
      "1": "#A4B9FA",
      "-1": "#e53935"
    }
  }
}
```

## Пример: обрыв линии на нулевых точках

Если в данных есть точки без значений (значение `null`), Chart.js по умолчанию может соединять соседние точки через пропуск. Чтобы линия **обрывалась** в таких местах, а не проходила по нулям, задайте:

```json
{
  "dataset": {
    "spanGaps": false
  }
}
```

Со значением `spanGaps: false` Chart.js не рисует отрезок к точке и от точки, если её значение равно `null` — линия обрывается, а не идёт по нулю.

> **Важно:** обрыв происходит только для значений `null` (отсутствие данных), а не для числового нуля `0`. Если источник данных (отчёт) возвращает `0` вместо `null` для пустых ячеек, линия будет проведена через точку с нулевым значением. Убедитесь, что отчёт возвращает `null` для ячеек без данных.

## Ограничения

- Максимальная длина строки JSON — 8000 символов.
- Функции и выражения JavaScript не поддерживаются — только статический JSON.
- При несовпадении количества элементов dataset'ов и `pointColors` лишние ключи игнорируются.
- Настройки применяются после общих параметров панели (вкладка «Общие настройки»), поэтому имеют приоритет над ними.
