# Взаимодействие с Точка Банком при организации платежа (PHP-бэкенд)

## Обзор

Платёжная интеграция с Точка Банком реализована в виде двухшагового API-процесса, который управляется системой отчётов PHP-бэкенда CRM. Ниже описаны шаги взаимодействия бэкенда с точкой при пополнении баланса.

---

## Архитектура

PHP-бэкенд реализован в `index.php` и построен на универсальном движке отчётов. Каждый отчёт имеет числовой идентификатор (`id`) и хранится в базе данных. Когда клиентское приложение делает HTTP-запрос вида `/my/report/<id>?JSON`, PHP:

1. Загружает определение отчёта из БД по идентификатору.
2. Формирует SQL-запрос с учётом параметров из `$_REQUEST`.
3. Если в запросе есть параметр `confirmed` **и** у отчёта есть колонки `SET`/`UPDATE` — выполняет модификацию данных (INSERT/UPDATE/DELETE).
4. Отдаёт результат в JSON-формате согласно переданному режиму (`JSON`, `JSON_KV`, `JSON_CR` и т.д.).

---

## Шаги взаимодействия при проведении платежа

### Шаг 1 — Создание платёжного поручения (отчёт 2192)

**HTTP-запрос от JS-клиента:**
```
POST /my/report/2192?JSON&confirmed=1&amount=<сумма>
Content-Type: application/x-www-form-urlencoded

_xsrf=<токен>
```

**Что делает PHP-бэкенд:**

1. `isApi()` возвращает `true` (параметр `JSON` присутствует в GET-строке).
2. Движок отчётов загружает определение отчёта `2192` из базы данных.
   - Отчёт `2192` содержит SQL-логику создания платёжного поручения во внутренней системе CRM.
   - Параметр `amount` передаётся как фильтр и подставляется в запрос.
3. Поскольку в запросе присутствует параметр `confirmed=1`, PHP выполняет UPDATE/INSERT операцию — создаёт запись о платёжном поручении в базе данных CRM.
4. При этом также инициируется запрос к API Точка Банка для регистрации заказа (детали хранятся в логике отчёта `2192` в БД).
5. Ответ возвращается в JSON-формате. При успехе HTTP-статус `200 OK`.

**Результат:** В базе данных CRM создано платёжное поручение, связанное с суммой пополнения.

---

### Шаг 2 — Получение платёжной формы Точка Банка (отчёт 1905)

**HTTP-запрос от JS-клиента:**
```
GET /my/report/1905?JSON_KV
```

**Что делает PHP-бэкенд:**

1. `isApi()` возвращает `true` (параметр `JSON_KV` начинается с `JSON`).
2. Движок отчётов загружает определение отчёта `1905` из базы данных.
   - Отчёт `1905` выполняет SQL-запрос, который обращается к данным созданного на шаге 1 платёжного поручения.
   - В данных отчёта содержится HTML-форма платёжной страницы Точка Банка (поле `Pay`).
3. Режим вывода `JSON_KV` формирует JSON-массив объектов — по одному объекту на строку результата, где ключи — заголовки колонок, значения — данные строки.
4. Ответ:
   ```json
   [
     {
       "Pay": "<form action=\"https://...tochka.../pay\" method=\"POST\">...</form>"
     }
   ]
   ```

**Результат:** JS-клиент получает готовую HTML-форму для редиректа на платёжную страницу Точка Банка, внедряет её в DOM и вызывает `form.submit()`, что перенаправляет пользователя на Точка Банк.

## Схема последовательности

```
Браузер          JS-клиент          PHP-бэкенд (index.php)      БД CRM        Точка Банк
   |                 |                        |                    |               |
   | [клик "Оплатить"] |                      |                    |               |
   |---------------->|                        |                    |               |
   |                 | POST /my/report/2192?  |                    |               |
   |                 | JSON&confirmed=1&      |                    |               |
   |                 | amount=<N>             |                    |               |
   |                 |----------------------->|                    |               |
   |                 |                        | SELECT + INSERT/   |               |
   |                 |                        | UPDATE (отчёт 2192)|               |
   |                 |                        |------------------->|               |
   |                 |                        |       OK           |               |
   |                 |                        |<-------------------|               |
   |                 |      200 OK (JSON)     |                    |               |
   |                 |<-----------------------|                    |               |
   |                 |                        |                    |               |
   |                 | GET /my/report/1905?   |                    |               |
   |                 | JSON_KV                |                    |               |
   |                 |----------------------->|                    |               |
   |                 |                        | SELECT (отчёт 1905)|               |
   |                 |                        | -> HTML-форма Pay  |               |
   |                 |                        |------------------->|               |
   |                 |                        |  [{Pay: "<form>"}} |               |
   |                 |                        |<-------------------|               |
   |                 | [{Pay: "<form>"}]      |                    |               |
   |                 |<-----------------------|                    |               |
   |                 |                        |                    |               |
   |  form.submit()  |                        |                    |               |
   |<----------------|                        |                    |               |
   |                 |                        |                    |               |
   |    [POST/redirect → Точка Банк]          |                    |               |
   |-------------------------------------------------------------->|               |
   |                                          |                    |               |
   |               [страница оплаты Точка Банк]                    |               |
   |<--------------------------------------------------------------|               |
```

## Параметры запросов

### Отчёт 2192 (создание поручения)

| Параметр   | Тип         | Обязательный | Описание                                      |
|------------|-------------|:------------:|-----------------------------------------------|
| `JSON`     | GET-флаг    | да           | Переключает бэкенд в API-режим                |
| `confirmed`| GET-флаг    | да           | Разрешает выполнение UPDATE/INSERT в отчёте   |
| `amount`   | GET-параметр| да           | Сумма пополнения в рублях                     |
| `_xsrf`    | POST-тело   | да           | XSRF-токен для защиты от CSRF-атак            |

### Отчёт 1905 (получение формы)

| Параметр   | Тип      | Обязательный | Описание                                          |
|------------|----------|:------------:|---------------------------------------------------|
| `JSON_KV`  | GET-флаг | да           | Режим вывода: массив объектов `{colName: value}`  |

---

## Режимы JSON-вывода движка отчётов

Движок отчётов `index.php` поддерживает несколько форматов API-ответа:

| Параметр    | Формат ответа                                                  |
|-------------|----------------------------------------------------------------|
| `JSON`      | `{"columns": [...], "data": [...], "header": "..."}` (полный) |
| `JSON_KV`   | `[{"col1": val1, "col2": val2, ...}, ...]` (по строкам)       |
| `JSON_CR`   | `{"columns": [...], "rows": [...], "totalCount": N}`          |
| `JSON_DATA` | `{"col1": val1, ...}` (только первая строка, плоский объект)  |

---

## Безопасность

- **XSRF-защита:** Параметр `confirmed` в сочетании с обязательным `_xsrf`-токеном в теле POST-запроса защищает операцию создания поручения от межсайтовых подделок запросов.
- **Сессионная аутентификация:** Все запросы отправляются с `credentials: 'include'`, что означает передачу сессионных cookie. PHP-бэкенд проверяет авторизацию пользователя перед выполнением отчёта.
- **Проверка суммы:** JS-клиент валидирует `amount > 0` перед отправкой запроса, а бэкенд обрабатывает параметр через движок фильтров отчётов.
