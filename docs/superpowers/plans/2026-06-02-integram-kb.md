# База знаний Integram (`docs/kb/`) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (рекоменд.) / executing-plans, task-by-task. Steps — чекбоксы `- [ ]`.

**Goal:** Разрезать монолит `docs/integram-app-workflow.md` (2566 строк) на тематические доки `docs/kb/*` по шаблону «человек→агент→грабли», собрать README-индекс (каталог + сводный индекс граблей + правило дополнения), ужать монолит до обзора+ссылок.

**Architecture:** Markdown-доки в `docs/kb/`. Источник переноса — секции `integram-app-workflow.md` (карта ниже). Существующие сильные доки (integram-reports.md, StructureRules.md, MCP.md, UPDATE_SCRIPT.md) НЕ копируем — ссылаемся. Каждый док по единому шаблону. «Тесты» = проверка структуры (маркеры шаблона) + рабочие относительные ссылки + контент не потерян.

**Tech Stack:** Markdown. Проверки — `grep`/ручная сверка ссылок. Worktree `/tmp/crm-kb`, ветка `kb-docs` от origin/main. PR в ideav/crm.

**Спека:** `docs/superpowers/specs/2026-06-02-integram-kb-design.md`.

## Шаблон каждого kb-дока (СТРОГО)
```markdown
# <Тема>
> Часть базы знаний Integram. Индекс: [docs/kb/README.md](README.md)

## На пальцах (для человека)
5–10 строк: что это, когда нужно, идея. Без команд.

> ────────── дальше — справочник для агента ──────────

## Операции (для агента)
Точные вызовы: метод/путь/параметры/пример/ответ.

## Грабли
- **Симптом:** … → **Причина:** … → **Фикс:** … *(дата, PR/issue)*
```
Маркеры для проверки: строки `## На пальцах (для человека)`, `дальше — справочник для агента`, `## Грабли`.

## Task 2: `schema.md` (DDL `_d_*`)

**Files:** Create `docs/kb/schema.md`. Source: integram-app-workflow.md секции 2.1/2.2/2.5/3.1–3.4/3.6/3.7 + «Метаданные справочник».

- [ ] **Step 1: Написать `schema.md`** по шаблону:
  - «На пальцах»: схема = таблицы(типы)+реквизиты; два вида связей (ссылка-справочник vs подчинённая) — самая частая дорогая ошибка; когда что.
  - «Операции»: создать таблицу `_d_new t=3 val`; типы колонок (t=3/8/9/4/13/14/11/12/10); добавить колонку (`_d_new`→`_d_req`→`_d_alias`); ссылка (`_d_ref`+`_d_req`); подчинённая (`_d_req/{parent} t={childType}` + поле дочернему → регистрируется как arr); переименование (`_m_save/{tableId} t{base}`, `_d_save` для реквизита; orig vs id!); удаление реквизита `_d_del_req/{id} forced=1`; удаление таблицы `_d_del/{id}` (только пустой); **удаление таблицы-бывшего-справочника**: ref-тип в метаданных как `referenced` → `_d_del/{referencedId}`, снять гранты ролей (таб.116) и колонки отчётов (таб.28), ПОТОМ дроп; маски DIR_MASK/FILE_MASK; метаданные (reqs[].id vs .orig, arr_id, ref, attrs/alias). Примеры curl.
  - «Грабли»: `_d_alias на id таблицы → metadata?JSON битый, чинить _d_save/{tableId} val=Имя&t=3`; `_d_req возвращает {obj:tableId, id:reqId} — брать id`; `снять ссылочный реквизит раньше удаления записей-владельцев → осиротевшие экземпляры, тип не удалить`; `_d_new идемпотентен — на дубль вернёт существующий id в obj`.
- [ ] **Step 2: Проверка** — маркеры шаблона (≥3); ссылка на StructureRules.md; команды совпадают с фактами из feedback_integram_schema_d_commands/integram_dir_admin_file_api (память) и кода.
- [ ] **Step 3: Commit** `docs(kb): schema.md — DDL _d_* (таблицы/связи/удаление/метаданные)`.
