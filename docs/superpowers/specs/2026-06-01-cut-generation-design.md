# Подзадача D3b: генерация резок из необеспеченных позиций

**Дата:** 2026-06-01
**Источник:** ideav/atex#52 («распределять по резкам … автоматически сгенерированный
план»). Финал эпика — генерация Производственных резок под необеспеченные позиции заказов.
**Где:** `download/atex/js/production-planning.js` (Диспетчер). Стек на D3c (#3076).
**Эпик:** [[atex_epic52_planning]]. Готово: A,B,C,E,D1,D2,D3a (merged), D3c #3076.

## Контекст и решения (согласованы)

Кнопка «Сгенерировать резки»: для позиций заказов без «Обеспечения» создаёт
Производственные резки. Решения:
- **Тип резки — подбирать** по сырью+ширине (как C); если подходящего типа нет — позицию
  **пропустить** (с предупреждением).
- **Кол-во резок** = `ceil(заказанное_кол-во / роликов_в_резке)` (ролики-в-резке из типа резки).
- **Поля резки — авто:** статус «Запланирована», подобранный тип, **авто-станок** (не из
  стоп-листа E, с балансировкой загрузки) + **FIFO-партия** сырья (старейшая с остатком).
- На каждую созданную резку — запись «Обеспечение» (позиция → резка).

Интегрирует: C (подбор типа), B (ролики/ширины), E (стоп-лист станка), A (остаток партии/FIFO).

## Чистое ядро генерации (тестируемое) — центр D3b

В ядре `planning` (`production-planning.js`). Вход — дескрипторы (плумбинг их соберёт):
- `positions`: `[{id, materialId, width, qty}]`
- `supplies`: `[{positionId}]` (из cut_planning)
- `cutTypeIndex`: `{ typeId: { materialId, widths:[{width, qty}] } }` (тип → сырьё + полосы)
- `slitters`: `[{id, stopMaterialIds:[…]}]` (E)
- `batches`: `[{id, materialId, dateKey, remainder}]` (FIFO)

Функции:
- `unsuppliedPositions(positions, supplies)` → позиции без обеспечения (id не в supplies).
- `matchCutType(cutTypeIndex, materialId, width)` → typeId подходящего типа (сырьё совпадает И
  ширина среди полос; при нескольких — с наибольшим qty этой ширины; детерминированно) или null.
- `rollersPerCut(cutTypeIndex, typeId, width)` → qty полосы этой ширины в типе (роликов за резку);
  0 если нет.
- `cutsNeeded(qty, perCut)` → `perCut>0 ? max(1, ceil(qty/perCut)) : 0`.
- `pickSlitter(slitters, materialId, loadBySlitterId)` → id станка, не запрещающего сырьё
  (`isMaterialBlocked`), с минимальной текущей загрузкой `loadBySlitterId` (балансировка);
  null если все запрещают.
- `pickBatchFIFO(batches, materialId)` → старейшая (min dateKey) партия этого сырья с
  `remainder>0`; null если нет.
- `generateCutPlan(input)` → `{ plan:[{positionId, cutTypeId, slitterId, batchId}], skipped:[{positionId, reason}] }`:
  для каждой необеспеченной позиции — `matchCutType` (нет → skipped «нет типа»); `rollersPerCut`+
  `cutsNeeded` (0 → skipped); подобрать слиттер (балансировка — вести счётчик загрузки по ходу) и
  FIFO-партию; добавить `cutsNeeded` записей в plan. Чистая, детерминированная, вход не мутирует.

## Плумбинг данных (контроллер)

- **positions_list** (live) += колонка `position_material_id` (abn_ID Позиция→Вид сырья) — чтобы
  знать сырьё позиции. (`position_width`/`position_qty` уже есть.)
- **`unsupplied`**: позиции из `positions_list`, чьи id не встречаются в `supply_position_id`
  отчёта `cut_planning` (уже грузится).
- **cutTypeIndex**: загрузить типы резки выбранных сырьёв и их «Полоса» (ширина+кол-во) —
  по образцу C (object/{Тип резки} по сырью + object/{Полоса}?F_U). Лениво по сырью необеспеченных
  позиций (ограниченный набор).
- **batches с FIFO**: расширить загрузку партий — `materialId`, дата прихода (→ dateKey), остаток
  (м² или м). (`loadBatchMaterialMap` уже даёт material; добавить дату+остаток.)
- **slitters**: уже `[{id, stopMaterialIds}]` (E). `loadBySlitterId` — текущее число резок на
  станок (из `this.cuts`).

## Кнопка «Сгенерировать резки» (контроллер)

Рядом с «Создать резку»/«Запланировать». По клику:
1. модалка подтверждения (не нативный confirm): «Создать резки под N необеспеченных позиций?».
2. собрать входы (positions/supplies/cutTypeIndex/slitters/batches) → `generateCutPlan`.
3. для каждой записи plan: `_m_new/{Производственная резка}` (Тип резки, Слиттер, Партия сырья,
   Статус «Запланирована») → получить cutId → `_m_new/{Обеспечение}` `up={positionId}` со ссылкой
   на резку (как `createSupply`). Последовательно; `setBusy`.
4. `reload()`; нотификация «Создано N резок, пропущено M позиций (нет типа/данных)».
Переиспользовать паттерны `createCut`/`createSupply`. Стоп-лист уже учтён в `pickSlitter`.

## Критерии приёмки / тесты (ядро — node-тестами)

- `unsuppliedPositions`: исключает позиции с обеспечением.
- `matchCutType`: сырьё+ширина → тип; нет совпадения → null; выбор детерминирован.
- `rollersPerCut`/`cutsNeeded`: ceil, min 1, 0 при отсутствии.
- `pickSlitter`: пропускает запрещающие сырьё (E), балансирует по загрузке; все запрещают → null.
- `pickBatchFIFO`: старейшая с остатком; нет → null.
- `generateCutPlan`: интеграция — позиции без типа в skipped; ceil-кол-во; станок не из стоп-листа;
  детерминизм; вход не мутируется.
- Кнопка/создание — DOM, проверяется чтением/фикстурой.
