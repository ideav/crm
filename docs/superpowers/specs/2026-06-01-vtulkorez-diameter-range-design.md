# Спецификация: диапазон диаметров втулкореза + авто-подбор задания

Дата: 2026-06-01
Приложение: atex/ateh (Integram, репозиторий ideav/crm)
Источник задачи: доработка по `ideav/atex/Bus Req Doc.txt` — «3 втулкореза под разные диаметры».

## Цель

Сейчас у втулкореза один диаметр («Диаметр, мм»). Нужно задать **диапазон**
диаметров втулок (min и max), с которыми он работает, и при создании задания
на втулки **автоматически подбирать** подходящий втулкорез по диаметру задания.

## Текущее состояние (как есть)

- Таблица **«Втулкорез»** (live id 1071): поля «Диаметр, мм» (req 1094, число) и
  «Статус» (req 1095). Записей 6: «Втулкорез 1/2/3» (диаметры 25/40/76) и
  «TC-20/40/76» (20/40/76).
- Таблица **«Задание на втулки»**: «Кол-во план», ссылка «Втулкорез» (→1071),
  «Диаметр, мм», «Кол-во факт», «Статус».
- Задания создаются и ведутся только на рабочем месте **«Пульт втулкореза»**
  (`download/atex/js/sleeve-cutter.js`, роль Оператор). У каждого задания есть
  выпадающий список втулкорезов — оператор выбирает любой из всех.
- Текущая форма **не читает** диаметр самого втулкореза (грузит втулкорезы как
  `{id, label}`). Это важно для безопасности миграции (см. ниже).

## Решения (согласовано)

1. **Модель диапазона** — два числовых поля min/max (не мульти-справочник).
2. **Назначение** — авто-подбор: система подставляет подходящий втулкорез по
   диаметру задания; оператор может переопределить.
3. **Где задаются диапазоны** — в админке Integram (правка записей «Втулкорез»),
   без нового UI.

## Дизайн

### 1. Изменение схемы БД (таблица «Втулкорез» 1071)

- Добавить **«Диаметр min, мм»** (число) и **«Диаметр max, мм»** (число).
- Удалить **«Диаметр, мм»** (req 1094) — *после* проверки, что поле не
  используется в live-отчётах/дашбордах. Если используется — поле оставляем,
  но в логике не задействуем (фиксируем в плане).
- «Статус» — без изменений.

**Миграция значений (live):** для каждой из 6 записей выставить
`min = max = текущий диаметр` (degenerate-диапазон). Это не меняет фактическое
поведение подбора (точное совпадение) и не навязывает бизнес-диапазоны на
проде. Реальные диапазоны выставляет админ/диспетчер в админке.
Для удобства — заготовленные команды на диапазоны из ТЗ (маленький 20–25,
средний 26–40, большой 41–76) прилагаются к задаче; применяются по желанию.

### 2. Пульт втулкореза (`sleeve-cutter.js`) — авто-подбор

- В картах имён добавить `CUTTER_REQ = { diaMin: 'Диаметр min, мм', diaMax: 'Диаметр max, мм' }`.
- `loadCutters` читает min/max каждого втулкореза → объект
  `{ id, label, diaMin, diaMax }`.
- Чистая функция ядра **`pickCutter(diameter, cutters)`**:
  - кандидаты — те, у кого `diaMin ≤ diameter ≤ diaMax` (границы включительно;
    пустой min трактуется как −∞, пустой max как +∞);
  - при нескольких кандидатах — с самым **узким** диапазоном (точная подгонка),
    при равной ширине — первый по порядку (детерминированно);
  - нет кандидатов → `null`.
- Поведение в карточке задания:
  - при изменении «Диаметр, мм» вызывается `pickCutter`;
  - `cutterId` подставляется **только** если поле пустое **или** прежнее значение
    было выставлено авто-подбором и больше не подходит под новый диаметр
    (ручной выбор оператора не перетирается);
  - в выпадающем списке — все втулкорезы (для переопределения), в подписи —
    диапазон: «Втулкорез 1 (20–25 мм)»;
  - нет подходящего → поле пустое + подсказка «нет втулкореза под Ø<d>».
- Признак «авто-выбран» хранится на задании в рантайме (`task.cutterAuto`),
  чтобы отличать авто-подстановку от ручного выбора. В БД не пишется.

### 3. Фикстура и тесты

- `docs/atex_metadata.json`: у таблицы «Втулкорез» (102) заменить req
  «Диаметр, мм» на «Диаметр min, мм» и «Диаметр max, мм».
- `experiments/atex-sleeve-cutter.test.js`: юнит-тесты `pickCutter`:
  - диаметр внутри диапазона → этот втулкорез;
  - несколько покрывающих → самый узкий;
  - нет покрытия → null;
  - границы включительно (min и max);
  - правило не-перетирания ручного выбора.
- Сид/примерные данные — диапазоны 20–25 / 26–40 / 41–76 для трёх втулкорезов.

### 4. Деплой

- Схема меняется на боевой БД через API (`_d_*`), форму деплоит Андрей через
  `update.php` (синхронно со схемой).
- Безопасность окна до деплоя: текущая развёрнутая форма не читает диаметр
  втулкореза, поэтому добавление полей min/max и удаление «Диаметр, мм» её не
  ломают. Новый код ищет поля по именам в метаданных.
- PR в `ideav/crm` через форк `unidel2035` (токен gaveron18 не проходит).

## Вне объёма (YAGNI)

- UI управления втулкорезами в atex (диапазоны — в админке).
- Жёсткий фильтр/блокировка несоответствующих втулкорезов (выбран авто-подбор
  с возможностью переопределения).
- Учёт статуса втулкореза при подборе (подбор по диапазону; статус —
  операционное состояние, не влияет на пригодность по диаметру).

## Открытые вопросы

- Финальные диапазоны для боевых записей — на усмотрение админа (миграция
  ставит min=max=текущий, реальные значения выставляются в админке).
