# Спецификация: вывод DATETIME «Партии ГП» как дата+время

Дата: 2026-06-01
Приложение: atex/ateh (Integram, ideav/crm)

## Проблема

Главное значение таблицы «Партия ГП» (1081) — поле типа **DATETIME (type 4)**,
которое приходит из API как **unix-штамп в секундах** (например `1777815000`).
В рабочем месте «Склад» (`warehouse.js`) оно выводится сырым числом вместо даты.

## Решение

1. **Общий форматтер** `formatDateTime(value)` в общем модуле `ref-search.js`
   (`AtexRefSearch.formatDateTime`):
   - пусто/`null` → `''`;
   - строка только из цифр (unix-штамп, секунды) → `new Date(n*1000)` →
     **`ДД.ММ.ГГГГ ЧЧ:ММ`** в локальном времени;
   - иначе (уже дата-строка / не число) → вернуть как есть (passthrough).
2. Применить в `warehouse.js` там, где штамп виден пользователю:
   - колонка «Дата прихода» в таблице партий;
   - подпись партии в выпадающем выборе.
3. **Не трогать** `arrivalKey` (FIFO-сортировка) — ей нужен сырой числовой штамп.
4. **Методичка** `docs/integram-app-workflow.md`: правило — поля DATETIME (type 4)
   приходят как unix-штамп; при выводе форматировать через
   `AtexRefSearch.formatDateTime`, чтобы не повторять ошибку.

## Вне объёма

- production-planning и dashboards штамп «Партии ГП» не выводят (ссылку хранят,
  но показывают по статусу/метражу/адресу) → правок не требуют. Форматтер доступен
  им, если позже понадобится.

## Тесты

Юнит-тест `formatDateTime`: unix-штамп → `ДД.ММ.ГГГГ ЧЧ:ММ` (TZ=UTC для
детерминизма), пусто → `''`, `null` → `''`, дата-строка → как есть, мусор → как есть.
