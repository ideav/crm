# Инструкция для человека-тестировщика Atex для #3029

**Задача:** [#3029](https://github.com/ideav/crm/issues/3029)
**Цель:** дать ручному тестировщику короткий воспроизводимый проход по
производственной цепочке Atex: поступление материалов, ввод заказа,
распределение заказа по исполнителям, создание нового типа резки, ввод брака,
назначение на втулкорез и оприходование остатка на склад.

Сценарий использует метку `АТХ-3029`. Для повторных прогонов добавляйте дату или
номер запуска, например `АТХ-3029-2026-05-31-01`, чтобы записи было легко найти
в списках и дашбордах.

## Участники и рабочие места

| Роль | Пользователь | Рабочее место |
|---|---|---|
| Менеджер | `manager` | `/atex/orders` |
| Диспетчер | `dispatcher` | `/atex/cut-calc`, `/atex/production-planning` |
| Оператор | `operator` | `/atex/slitter`, `/atex/sleeve-cutter` |
| Кладовщик | `operator` | `/atex/intake`, `/atex/warehouse` |
| Руководитель | `director` | `/atex/dashboards` |

Если в базе нет справочных значений из раздела ниже, создайте их заранее через
штатные таблицы Интеграма под администратором.

## Поступление материалов

1. Войти пользователем склада и открыть `/atex/intake`.
2. Нажать создание новой партии сырья.
3. Заполнить:
   - `Партия сырья: RM-АТХ-3029-01`;
   - `Вид сырья: Jumbo Thermal 910/4000 #3029`;
   - `Дата прихода: 2026-05-31`;
   - `Получено, м²: 3640`;
   - `Остаток, м²: 3640`.
4. Сохранить партию.
5. Проверить, что партия появилась в списке FIFO и остаток равен полученному
   количеству.

## Ввод заказа

1. Войти менеджером и открыть `/atex/orders`.
2. Нажать `Создать заказ`.
3. Выбрать клиента `ООО Ромашка-Термолента #3029`.
4. Указать статус `Новый`.
5. В поле `Лидер` указать `Диспетчер АТХ-3029`.
6. В `Примечания` ввести:
   `АТХ-3029: требуется новая резка 57x10+40x8, распределить на SL-3029 и TC-3029`.
7. Сохранить заказ и записать его номер.
8. Если система уже позволяет выбрать новый тип резки, сразу добавить позицию.
   Если типа ещё нет, вернуться к позиции после следующего раздела и заполнить:
   `Кол-во: 10`, `Вид сырья: Jumbo Thermal 910/4000 #3029`,
   `Тип резки: TT-АТХ-3029 57x10+40x8`, `Ширина, мм: 57`,
   `Длина, м: 1200`, `Диаметр втулки: 76`, `Статус: В работе`.
9. Перевести заказ в статус `Согласован`.

Ожидаемый результат: заказ с меткой `АТХ-3029` виден в списке, у него есть одна
позиция с новым типом резки и статусом `В работе`.

## Создание нового типа резки

1. Войти диспетчером и открыть `/atex/cut-calc`.
2. Нажать `Новый`.
3. Ввести название `TT-АТХ-3029 57x10+40x8`.
4. Выбрать `Вид сырья: Jumbo Thermal 910/4000 #3029`.
5. Указать `Ширина входа, мм: 910` и `Допуск, мм: 25`.
6. Ввести первую полосу: `Ширина, мм: 57`, `Количество: 10`,
   `Назначение: Заказ`.
7. Нажать `+ Добавить полосу`.
8. Ввести вторую полосу: `Ширина, мм: 40`, `Количество: 8`,
   `Назначение: Заказ`.
9. Проверить расчёт `Итого ножей = 18` и `Остаток, мм = 20`.
10. Нажать `Сохранить`.
11. Вернуться в `/atex/orders`, если позиция заказа ещё не была добавлена, и
    добавить её с типом резки `TT-АТХ-3029 57x10+40x8`.

Ожидаемый результат: тип резки и обе полосы сохранены, расчёт не меняется после
обновления страницы.

## Распределение заказа по исполнителям

В текущих рабочих местах Atex исполнители фиксируются через производственные
ресурсы: слиттер в `Производственная резка` и втулкорез в `Задание на втулки`.
На этом шаге распределите позицию заказа на слиттер.

1. Открыть `/atex/production-planning` под диспетчером.
2. Создать производственную резку:
   - `Слиттер: SL-3029`;
   - `Тип резки: TT-АТХ-3029 57x10+40x8`;
   - `Партия сырья: RM-АТХ-3029-01`;
   - `Дата план: 2026-06-01`;
   - `Статус: В очереди`;
   - `Примечания: АТХ-3029 распределено на SL-3029`.
3. В блоке обеспечения выбрать позицию заказа с меткой `АТХ-3029`.
4. Указать `Метраж, м: 1200`, статус `Зарезервировано`.
5. Нажать `Привязать к резке`.
6. Вернуться в `/atex/orders` и перевести заказ в статус `В производстве`.

Ожидаемый результат: в очереди по `SL-3029` есть резка с нужным типом, а запись
`Обеспечение` связывает позицию заказа и производственную резку со статусом
`Зарезервировано`.

## Ввод брака

1. Открыть `/atex/slitter` под оператором.
2. Выбрать резку `АТХ-3029` по слиттеру `SL-3029`.
3. Последовательно перевести статус `Ожидает` -> `Наладка` -> `В работе`.
4. Заполнить:
   - `Счётчик нач.: 1000`;
   - `Счётчик кон.: 2200`;
   - `Погонаж факт, м: 1200`;
   - `Брак, м²: 12`.
5. Добавить расход сырья по партии `RM-АТХ-3029-01` на `1200 м²`.
6. Перевести резку в статус `Завершён`.

Ожидаемый результат: значение `Брак, м²: 12` сохраняется в резке, расход сырья
создаёт запись `Расход сырья`, а `Остаток, м²` у партии сырья уменьшается.

## Назначение на втулкорез

1. Открыть `/atex/sleeve-cutter` под оператором.
2. Выбрать позицию заказа `АТХ-3029`.
3. Нажать `+ Добавить задание`.
4. Заполнить:
   - `Втулкорез: TC-3029`;
   - `Диаметр, мм: 76`;
   - `Кол-во план: 10`;
   - `Кол-во факт: 10`;
   - `Статус: Ожидает`.
5. Сохранить задание.
6. Перевести статус задания `Ожидает` -> `В работе` -> `Готово`.

Ожидаемый результат: задание на втулки привязано к позиции заказа, выбран `TC-3029`,
сохранено `Кол-во факт: 10`, а сводка показывает выполнение `100%`.

## Оприходование остатка на склад

В этой проверке складской остаток после производства фиксируется как партия ГП.
Сырьевой остаток партии контролируется на шаге слиттера через уменьшение
`Остаток, м²`; готовый остаток для заказа приходуется на складе готовой
продукции.

1. Открыть `/atex/warehouse` пользователем склада.
2. Создать `Партия ГП` из завершённой резки `АТХ-3029`.
3. Заполнить:
   - `Ширина, мм: 57`;
   - `Кол-во рулонов: 10`;
   - `Метраж, м: 1200`;
   - `Адрес хранения: A-3029-01`;
   - `Статус: Есть`.
4. Сохранить партию.
5. Выполнить FIFO-привязку партии ГП к `Обеспечение` позиции заказа.
6. Перевести партию `Есть` -> `Зарезервирован` -> `Отгружен`.
7. Перевести обеспечение в `Выполнено`, позицию заказа в `Отгружена`, заказ в
   `Выполнен`.

Ожидаемый результат: партия ГП видна на складе по адресу `A-3029-01`, привязана
к обеспечению заказа, прошла статусы `Есть`, `Зарезервирован`, `Отгружен`.

## Финальная проверка

1. Открыть `/atex/dashboards` под руководителем.
2. Найти `АТХ-3029` в карточке пути продукции.
3. Проверить, что в одном пути видны: заказ, позиция, обеспечение,
   производственная резка, расход сырья, задание на втулки и партия ГП.
4. Сверить финальные статусы:

| Объект | Ожидаемый статус или значение |
|---|---|
| Заказ | `Выполнен` |
| Позиция заказа | `Отгружена` |
| Производственная резка | `Завершён` |
| Обеспечение | `Выполнено` |
| Брак | `12 м²` |
| Задание на втулки | `Готово`, факт `10` |
| Партия ГП | `Отгружен` |

## Критерии приёмки

- Поступление материалов выполнено через `/atex/intake`, партия сырья создана с
  `Получено, м²: 3640` и `Остаток, м²: 3640`.
- Заказ введён через `/atex/orders`, содержит метку `АТХ-3029`, клиента, лидера
  и позицию с новым типом резки.
- Новый тип резки создан через `/atex/cut-calc`, расчёт показывает
  `Итого ножей = 18` и `Остаток, мм = 20`.
- Распределение заказа по исполнителям выполнено через `/atex/production-planning`:
  позиция привязана к резке на `SL-3029`, обеспечение в статусе
  `Зарезервировано`.
- Брак введён через `/atex/slitter`, значение `Брак, м²: 12` сохраняется после
  обновления страницы.
- Назначение на втулкорез выполнено через `/atex/sleeve-cutter`: выбран
  `Втулкорез: TC-3029`, статус дошёл до `Готово`, `Кол-во факт: 10`.
- Оприходование остатка на склад выполнено через `/atex/warehouse`: создана
  партия ГП, указан `Адрес хранения`, партия прошла статусы `Есть` ->
  `Зарезервирован` -> `Отгружен`.
- `/atex/dashboards` показывает один связанный путь продукции для метки
  `АТХ-3029`.
