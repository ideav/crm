# Возможности дэшборда Интеграм для обучающих видео и статей

**Цель документа:** дать готовую основу для видео-уроков, статей и демонстрационных сценариев по рабочему месту «Дэшборд» в Интеграме.

**Аудитория:** пользователи, аналитики, руководители, администраторы Интеграма и авторы обучающих материалов.

---

## Короткое описание для первого экрана

Дэшборд Интеграм - это рабочее место для просмотра показателей, таблиц, графиков и сводных данных из базы Интеграма. Он собирает несколько отчетов и расчетных блоков на одном экране, позволяет фильтровать данные по периоду и значениям, переключать типы визуализации и быстро показывать пользователю итоговую картину без перехода между множеством таблиц.

В обучающих материалах дэшборд удобно объяснять как «витрину данных»: данные остаются в таблицах и отчетах Интеграма, а дэшборд показывает их в нужной для роли форме - руководителю, менеджеру, аналитику или оператору.

---

## Основные возможности

### 1. Несколько листов в одном дэшборде

Дэшборд может состоять из нескольких вкладок-листов. Каждый лист показывает отдельный срез: например, продажи, задачи, финансы, операционные показатели или аналитику по сотрудникам.

Что показать в уроке:

- переключение между вкладками;
- разделение одного дэшборда на тематические страницы;
- сценарий «у руководителя один вход, но несколько рабочих экранов».

### 2. Панели с показателями и таблицами

Лист состоит из панелей. Панель - это отдельный блок с данными: таблица, набор строк с показателями, график, сводная таблица или текстовое пояснение.

Панель может показывать:

- значения по периодам;
- строки показателей с иерархией;
- данные из сохраненного отчета;
- результаты формул;
- табличные данные с группами колонок;
- примечания к панели в Markdown.

Практический пример для статьи: «Одна панель показывает выручку по месяцам, вторая - количество лидов, третья - воронку этапов, четвертая - комментарии к плану».

### 3. Работа с периодами

В верхней панели дэшборда есть общий фильтр периода:

- дата начала;
- дата окончания;
- детализация: неделя, месяц или год;
- кнопка применения фильтра.

Это позволяет одной и той же модели дэшборда показывать разные временные срезы без пересборки отчета.

Что важно объяснить:

- период управляет загрузкой данных на листе;
- показатели пересчитываются после применения фильтра;
- один и тот же дэшборд можно использовать для ежедневного, месячного и годового анализа.

### 4. Поиск по строкам

На листе есть быстрый поиск. Он помогает найти нужный показатель, строку или элемент панели без ручной прокрутки.

Подходящий учебный сценарий:

1. Открыть большой дэшборд.
2. Ввести часть названия показателя.
3. Показать, как лишние строки скрываются и остается нужный блок.

### 5. Фильтры внутри панели

Каждая панель может иметь собственный фильтр. Он применяется только к выбранной панели и не меняет остальные блоки листа.

Поддерживаемые сценарии:

- выбрать одно или несколько значений;
- отфильтровать числовой диапазон;
- отфильтровать диапазон дат;
- сбросить фильтр;
- увидеть активный фильтр по состоянию иконки панели.

В статьях это удобно объяснять через сравнение: общий фильтр периода отвечает за весь лист, а фильтр панели - за локальный разбор конкретного графика или таблицы.

### 6. Источники данных из отчетов

Дэшборд берет данные из отчетов Интеграма и из значений, сохраненных в модели дэшборда. Отчет можно использовать как источник для таблицы или графика.

Ключевая идея для обучения:

- отчет отвечает на вопрос «какие данные выбрать»;
- дэшборд отвечает на вопрос «как эти данные показать пользователю».

Покажите цепочку:

1. Создать или открыть отчет.
2. Проверить колонки, фильтры и группировки.
3. Подключить отчет к панели.
4. Настроить визуализацию по колонкам отчета.

### 7. Формулы и расчетные значения

В дэшборде можно использовать формулы для расчетных ячеек и строк. Формула позволяет получить значение из другого показателя, отчета или периода и пересчитать итог прямо на экране.

Что можно вынести в отдельный урок:

- простая расчетная строка;
- ссылка на показатель;
- расчет по данным отчета;
- редактирование формулы через модальное окно;
- пересчет связанных ячеек после изменения значения.

Для аудитории без технического опыта формулы лучше объяснять через бизнес-задачи: «конверсия», «план-факт», «средний чек», «доля в общем объеме».

### 8. Ручное редактирование значений

Если показатель должен храниться как план, прогноз или корректировка, значение можно внести вручную прямо в ячейку дэшборда. После сохранения оно используется в расчетах и отображении.

Подходящие примеры:

- план продаж на месяц;
- целевой KPI;
- ручная корректировка прогноза;
- комментарий или уточнение к отчетному периоду.

### 9. Типы визуализации

Панель может отображаться не только таблицей. Для панели доступны варианты:

| Тип | Когда использовать |
| --- | --- |
| Таблица | Детальный просмотр значений и строк |
| Линейный график | Динамика во времени |
| Круговая диаграмма | Доли категорий в одном периоде |
| Столбчатая диаграмма | Сравнение категорий и периодов |
| Диаграмма с областями | Накопленная или сравнительная динамика |
| Пузырьковая диаграмма | Сравнение трех числовых параметров |
| Диаграмма-воронка | Последовательные этапы процесса |
| Сводная таблица | Пересечение строк, колонок и значений |

В видео полезно показать одну и ту же панель в нескольких режимах: сначала таблица, затем линейный график, затем столбчатая диаграмма или воронка.

### 10. Настройка полей для графиков

Для визуализаций можно назначать поля:

- ось X;
- значение;
- серии;
- стек;
- строки, колонки и значения для сводной таблицы;
- X, Y и размер для пузырьковой диаграммы;
- этап и значение для воронки.

Если источник - отчет, настройка строится по колонкам отчета. Если источник - табличная панель, дэшборд может использовать строки и колонки самой панели.

### 11. Настройки внешнего вида графиков

В общих настройках панели доступны параметры отображения:

- высота панели и графика;
- ширина панели по адаптивной сетке из 12 колонок;
- толщина столбцов;
- цветовая гамма;
- размер подписей осей;
- поворот подписей оси X;
- позиция легенды или отключение легенды;
- количество знаков в подсказках;
- максимальная ширина панели.

Для продвинутых материалов есть отдельная тема: кастомные настройки Chart.js через JSON. Они позволяют точечно переопределять цвета, толщину линий, пунктир, параметры легенды и другие опции диаграмм.

См. также: [Кастомные настройки диаграммы](CHART_CUSTOM_OPTIONS.md).

### 12. Режим плитки и адаптивная компоновка

Дэшборд поддерживает плиточный режим: панели выстраиваются в сетку и могут занимать разную ширину. Это помогает собрать обзорный экран, где сразу видны несколько показателей.

Что показать:

- включение и выключение режима плитки;
- изменение ширины панели;
- сброс сохраненных размеров;
- поведение на широком экране и на мобильном размере.

В статье стоит отдельно подчеркнуть: плитка нужна для обзорных рабочих мест, а обычный вертикальный режим - для последовательного анализа.

### 13. Размеры таблиц и графиков

Пользователь может менять размеры отображения, а администратор - задавать устойчивые настройки панели. Это полезно, когда одна диаграмма должна быть крупной, а вспомогательная таблица - компактной.

Примеры:

- большая воронка на первом экране;
- компактные KPI-плитки;
- широкая сводная таблица;
- высокий график динамики.

### 14. Примечания к панели

К панели можно добавить текстовое примечание. Оно отображается рядом с данными и поддерживает простую Markdown-разметку: списки, выделение, ссылки и короткие пояснения.

Учебные сценарии:

- добавить пояснение к методике расчета;
- оставить инструкцию для пользователя;
- указать, откуда берутся данные;
- объяснить, почему показатель может отличаться от отчета бухгалтерии.

### 15. Роли пользователя и администратора

Обычный пользователь работает с готовым дэшбордом:

- выбирает период;
- применяет поиск;
- открывает фильтр панели;
- переключает доступные визуализации;
- меняет локальный вид, если это разрешено интерфейсом.

Администратор настраивает модель:

- структуру листов и панелей;
- источники данных;
- формулы;
- варианты визуализации;
- размеры и цветовые настройки;
- публикацию рабочего места в меню.

В обучении важно разделять эти два режима. Первые уроки лучше делать для пользователя, а отдельный блок - для администратора и аналитика.

## Рекомендуемая структура видео-уроков

### Урок 1. Что такое дэшборд и зачем он нужен

Цель: показать дэшборд как готовое рабочее место.

План:

1. Открыть дэшборд из меню.
2. Показать вкладки, панели и общий фильтр периода.
3. Переключить несколько визуализаций.
4. Объяснить, что данные берутся из таблиц и отчетов Интеграма.

Результат для зрителя: он понимает, где находится дэшборд и какую задачу решает.

### Урок 2. Период, поиск и локальные фильтры

Цель: научить пользователя быстро находить нужный срез.

План:

1. Изменить даты.
2. Переключить детализацию на месяц или год.
3. Найти показатель через поиск.
4. Открыть фильтр панели.
5. Применить и сбросить фильтр.

Результат: пользователь умеет самостоятельно смотреть нужный период и сегмент.

### Урок 3. Панели и источники данных

Цель: объяснить, из чего состоит дэшборд.

План:

1. Показать панель с показателями.
2. Показать панель на основе отчета.
3. Объяснить связь «таблица - отчет - дэшборд».
4. Разобрать, почему важно правильно настроить колонки отчета.

Результат: пользователь понимает, что дэшборд не хранит все данные сам, а визуализирует подготовленные источники.

### Урок 4. Графики и сводные таблицы

Цель: показать разные способы визуального анализа.

План:

1. Открыть настройки отображения панели.
2. Включить линейный график, столбцы и круговую диаграмму.
3. Назначить поля осей и значений.
4. Показать сводную таблицу.
5. Сохранить вариант по умолчанию.

Результат: администратор или аналитик умеет выбрать подходящую визуализацию для задачи.

### Урок 5. Формулы и ручные значения

Цель: показать расчетные возможности.

План:

1. Открыть расчетную ячейку.
2. Показать формулу.
3. Изменить исходное значение.
4. Проверить пересчет результата.
5. Объяснить примеры: план-факт, конверсия, доля, средний показатель.

Результат: зритель понимает, когда использовать отчет, а когда формулу или ручное значение.

### Урок 6. Компоновка рабочего места

Цель: подготовить дэшборд к ежедневному использованию.

План:

1. Включить плиточный режим.
2. Изменить ширину и высоту панелей.
3. Настроить легенду, подписи и цвета.
4. Добавить примечания.
5. Проверить вид на узком экране.

Результат: дэшборд выглядит как аккуратное рабочее место для конкретной роли.

### Урок 7. Публикация и поддержка

Цель: объяснить жизненный цикл дэшборда.

План:

1. Проверить корректность отчетов.
2. Проверить права доступа.
3. Добавить рабочее место в меню.
4. Показать пользователям, как читать показатели.
5. Собрать обратную связь и улучшить панели.

Результат: дэшборд готов не только технически, но и методически.

## Темы для статей

1. **Что такое дэшборд Интеграм и чем он отличается от отчета.** Объяснить разницу между хранением данных, запросом и визуальным рабочим местом.
2. **Как подготовить данные для дэшборда.** Рассказать про таблицы, связи, отчеты, фильтры и группировки.
3. **Как собрать первый дэшборд.** Пошагово разобрать вкладки, панели, период и источники.
4. **Как выбрать правильный тип диаграммы.** Сравнить таблицу, линию, столбцы, круг, область, пузырьковую диаграмму, воронку и сводную таблицу.
5. **Как настроить фильтры дэшборда.** Показать общий период, поиск и фильтры отдельных панелей.
6. **Как использовать формулы в дэшборде.** Разобрать расчетные показатели без глубокого погружения в код.
7. **Как оформить дэшборд для руководителя.** Показать плитки, размеры, легенду, цвета и примечания.
8. **Как проверять дэшборд перед публикацией.** Дать чек-лист источников, прав, периодов и визуального отображения.

## Демонстрационные сценарии

### Продажи

Панели:

- выручка по месяцам;
- новые лиды;
- конверсия по этапам;
- воронка продаж;
- план-факт по менеджерам;
- комментарии к отклонениям.

Что подчеркнуть: руководитель видит общую динамику и может быстро перейти к проблемному периоду или этапу.

### HR и обучение

Панели:

- кандидаты по этапам;
- количество интервью;
- принятые офферы;
- старт обучения;
- загрузка ответственных;
- причины отказов.

Что подчеркнуть: дэшборд показывает процесс как цепочку, а не как набор отдельных таблиц.

### Финансы

Панели:

- доходы и расходы;
- план-факт бюджета;
- задолженность;
- платежи по периодам;
- структура затрат;
- ручной прогноз.

Что подчеркнуть: часть данных можно получать из отчетов, а плановые значения вносить вручную.

## Чек-лист перед записью урока

- Дэшборд открывается по понятному пункту меню.
- Вкладки названы по задачам пользователя, а не по техническим источникам.
- На первом экране видны самые важные панели.
- Период по умолчанию показывает достаточно данных для демонстрации.
- В отчетах есть тестовые строки для графиков и фильтров.
- У каждой диаграммы понятные подписи осей и легенда.
- Лишние или пустые панели скрыты.
- Примечания объясняют сложные расчеты.
- Роли и права доступа проверены заранее.
- Для статьи подготовлены скриншоты: общий вид, фильтр панели, настройки графика, плиточный режим.

## Формулировки для анонсов и описаний

Короткий вариант:

> В этом уроке разбираем дэшборд Интеграм: как смотреть показатели по периодам, фильтровать панели, переключать графики и использовать дэшборд как ежедневное рабочее место руководителя или аналитика.

Расширенный вариант:

> Дэшборд Интеграм помогает собрать данные из таблиц и отчетов в один понятный экран. В уроке покажем вкладки, панели, период, локальные фильтры, графики, сводные таблицы, формулы и плиточную компоновку. После просмотра вы сможете читать готовый дэшборд и понимать, какие настройки нужны для создания собственного рабочего места.

Вариант для статьи:

> Дэшборд в Интеграме - это слой визуального анализа поверх таблиц и отчетов. Он не заменяет структуру данных, а делает ее удобной для конкретной роли: руководителя, менеджера, аналитика или оператора. Ниже разберем, из каких частей состоит дэшборд и какие возможности стоит показать в обучающих материалах.
