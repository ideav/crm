# Требования к окнам подсказок в рабочих местах

Этот документ определяет единые требования к реализации всплывающих окон подсказок (hints) во всех рабочих местах системы. Он основан на опыте реализации подсказок в workspace `upload` (PR #1269, PR #1272, PR #1273).

Общий механизм отображения подсказок вынесен в `/js/hints.js`. В файле рабочего места размещаются только тексты подсказок (HTML-структура) и привязка специфичных для рабочего места триггеров.

---

## 1. Позиционирование

- **Начальная позиция**: окно подсказки **всегда появляется по центру экрана** (горизонтально и вертикально), чтобы сразу привлечь внимание пользователя.
- Реализуется через CSS: `position: fixed; top: 50%; left: 50%; transform: translate(-50%, -50%);`
- После перетаскивания окна пользователем `transform` сбрасывается, и окно фиксируется в новых абсолютных координатах.

## 2. Внешний вид

- **Фон**: `var(--card-bg, #fff)` — соответствует теме оформления.
- **Скругление**: `border-radius: 12px`.
- **Тень**: заметная, но ненавязчивая — `box-shadow: 0 8px 40px rgba(0,0,0,0.28)`.
- **Ширина**: `90%`, максимум `420px`, чтобы выглядело аккуратно на любом экране.
- **Заголовок**: содержит **цветную иконку** (например, 💡) перед текстом — для визуального привлечения внимания.
- **Немодальность**: окно не перекрывает фон затемнением. Пользователь может одновременно читать подсказку и выполнять действия на странице.

## 3. Перетаскивание (desktop)

- Вверху окна размещается **видимая ручка для перетаскивания** (`hint-drag-handle`) высотой `2rem` с курсором `grab`.
- Ручка содержит **иконку** (например, `⠿⠿`) с атрибутом `title="Это окно можно перетаскивать по экрану"`, чтобы пользователю было очевидно, что окно можно двигать.
- Иконка отображается по центру ручки цветом `var(--text-secondary, #888)`.
- Пользователь может перетаскивать окно по экрану мышью.
- Окно остаётся **в пределах вьюпорта** (не выходит за границы экрана).
- При начале перетаскивания `transform` сбрасывается; окно переходит на абсолютные координаты `top`/`left`.

## 4. Мобильные устройства (≤767px)

- Перетаскивание на мобильных устройствах недоступно (нет поддержки mouse drag).
- Вместо этого отображается **круглая кнопка-тоггл** (↑/↓), которая перемещает окно подсказки вверх или вниз экрана.
- Кнопка-тоггл видима только на мобильных экранах (`display: none` по умолчанию, `display: flex` через `@media (max-width: 767px)`).

## 5. Прогрессия подсказок (event-driven)

- Следующая подсказка **появляется автоматически** при выполнении пользователем посоветованного действия — без кнопки «Далее».
- Каждая подсказка привязана к конкретному событию (клик на кнопку, переход к следующему шагу и т.д.).
- Закрытие текущей подсказки через кнопку «Закрыть» / «Понятно» завершает всю серию.

## 6. Кнопки управления

- **«Закрыть»** (промежуточные подсказки): стиль `outline` / нейтральный, цвет `var(--text-secondary)`.
- **«Понятно»** (финальная подсказка): стиль primary (`var(--button-primary, #0d6efd)`, белый текст).

## 7. Сохранение состояния (cookies)

- Факт просмотра подсказок сохраняется в cookie `hints_seen_workspaces` как список идентификаторов workspace через запятую (например, `upload,kanban`).
- Cookie хранится 365 дней.
- Если workspace уже есть в списке, подсказки не показываются при следующем открытии.
- Cookie `hints_mode=off` глобально отключает все подсказки.

## 8. Идентификаторы и структура HTML

Префикс `{workspace}-hint-` используется для всех элементов, чтобы избежать конфликтов имён между разными рабочими местами. Полная структура HTML и пример инициализации приведены в разделе 10.

## 9. CSS-переменные

Используйте CSS-переменные темы оформления:
- `var(--card-bg, #fff)` — фон окна
- `var(--text-primary)` — цвет заголовка
- `var(--text-secondary)` — цвет текста и второстепенных кнопок
- `var(--border-color)` — цвет рамок кнопок
- `var(--button-primary, #0d6efd)` — цвет primary-кнопки

## 10. Разделение общего и специфичного кода

Механизм отображения подсказок реализован в `/js/hints.js` и **не изменяется** при добавлении новых рабочих мест. В файл рабочего места помещается только то, что уникально для него:

- HTML-структура: `hint-box` с шагами и кнопками (тексты, заголовки, количество шагов).
- Привязка специфичных для рабочего места триггеров (в колбэке `onInit`).
- Специфичные CSS-правила для `#workspace-hint-box` (при необходимости).

### Подключение в шаблоне

```html
<!-- Подключить общий скрипт один раз на странице -->
<script src="/js/hints.js"></script>

<!-- HTML-структура подсказок (тексты и шаги — специфичны для рабочего места) -->
<div id="myworkspace-hint-box">
    <div class="hint-drag-handle" id="myworkspace-hint-drag-handle" title="Это окно можно перетаскивать по экрану"><span class="hint-drag-icon">⠿⠿</span></div>
    <div id="myworkspace-hint-1" style="display:none;">
        <h3><span style="margin-right:0.4rem;">💡</span>Заголовок первой подсказки</h3>
        <p>Текст подсказки...</p>
        <div style="display:flex; justify-content:flex-end;">
            <button onclick="myworkspaceHintClose()">Закрыть</button>
        </div>
    </div>
    <!-- ... другие шаги ... -->
</div>
<button id="myworkspace-hint-mobile-toggle" ...></button>

<!-- Инициализация: только специфичная для рабочего места логика -->
<script>
document.addEventListener('DOMContentLoaded', function() {
    window.initHints({
        workspace: 'myworkspace',
        steps: 3,
        onInit: function(api) {
            // Привязать триггеры, специфичные для данного рабочего места
            var btn = document.getElementById('some-action-btn');
            if (btn) btn.addEventListener('click', function() { api.advance(2); });
        }
    });
});
</script>
```

### API, предоставляемый `initHints`

После вызова `initHints` на `window` появляются следующие функции (на примере workspace `myworkspace`):
- `window.myworkspaceHintShow(n)` — показать шаг `n`.
- `window.myworkspaceHintClose()` — закрыть окно и сохранить в cookies.
- `window.myworkspaceHintAdvance(n)` — перейти к шагу `n`, если окно открыто.
- `window.myworkspaceHint` — объект с методами `show`, `close`, `advance`.
