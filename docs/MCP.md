# Справочник команд API Интеграма для MCP-сервера

## Базовые параметры

| Параметр | Где используется | Описание |
|----------|------------------|----------|
| `{db}` | Во всех URL | Имя базы данных (задаётся при создании) |
| `{id}` | В командах с ID | Числовой идентификатор объекта (таблицы, колонки, записи) |
| `token` | В теле POST-запросов | Токен авторизации из `auth` |
| `_xsrf` | В теле POST-запросов | XSRF-токен из `auth` |
| `JSON=1` | В URL всех запросов | Формат ответа |
| `JSON_DATA=1` | В `object` | Альтернативный формат ответа |

---

## 1. Управление базами данных

### Создать базу
```bash
POST https://integram.io/my/_new_db?JSON=1
Body: db={имя_базы}&template=ru
```
**Ответ:** `{ "status": "Ok", "id": 221756 }`

---

## 2. Авторизация

### Получить токены
```bash
POST https://integram.io/{db}/auth?JSON=1
Body: login={логин}&pwd={пароль}
```
**Ответ:** `{ "_xsrf": "...", "token": "...", "id": "632" }`

---

## 3. Управление таблицами (терминами)

### Создать таблицу
```bash
POST https://integram.io/{db}/_d_new?JSON=1
Body: t=3&val={имя_таблицы}&_xsrf={xsrf}&token={token}
```
**Ответ:** `{ "obj": 641 }` ← ID таблицы

### Удалить таблицу (только пустую)
```bash
POST https://integram.io/{db}/_d_del/{id_таблицы}?JSON=1
Body: _xsrf={xsrf}&token={token}
```
**Ответ:** `{ "id": 642 }`

### Получить список всех таблиц
```bash
GET https://integram.io/{db}/dict?JSON=1
Body: token={token}
```
**Ответ:** `{ "18": "Пользователь", "22": "Запрос", ... }`

### Получить метаданные таблицы (колонки, типы, ID реквизитов)
```bash
GET https://integram.io/{db}/metadata/{id_таблицы}?JSON=1
```
**Ответ:** 
```json
{
  "id": "641",
  "val": "Проект",
  "reqs": [
    { "id": 643, "val": "Название проекта", "type": "3" },
    { "id": 644, "val": "Бюджет", "type": "13" }
  ]
}
```

---

## 4. Управление колонками (реквизитами)

### Создать колонку (термин)
```bash
POST https://integram.io/{db}/_d_new?JSON=1
Body: t={тип}&val={имя_колонки}&_xsrf={xsrf}&token={token}
```
**Ответ:** `{ "obj": 642 }` ← ID колонки

**Типы колонок (параметр t):**
| Код | Тип |
|-----|-----|
| 3 | SHORT |
| 8 | CHARS |
| 9 | DATE |
| 4 | DATETIME |
| 13 | NUMBER |
| 14 | SIGNED |
| 11 | BOOLEAN |
| 12 | MEMO |
| 10 | FILE |
| 6 | PWD |

### Добавить колонку в таблицу
```bash
POST https://integram.io/{db}/_d_req/{id_таблицы}?JSON=1
Body: t={id_колонки}&_xsrf={xsrf}&token={token}
```
**Ответ:** `{ "id": 643 }` ← ID реквизита (используется для вставки данных!)

### Удалить колонку из таблицы (с данными)
```bash
POST https://integram.io/{db}/_d_del_req/{id_реквизита}?JSON=1
Body: forced=1&_xsrf={xsrf}&token={token}
```
**Ответ:** `{ "id": "641" }`

### Переместить колонку (изменить порядок)
```bash
POST https://integram.io/{db}/_d_ord/{id_реквизита}?JSON=1
Body: order={новый_порядок_с_0}&_xsrf={xsrf}&token={token}
```
При этом колонку 0 перемещать нельзя и на её место нельзя поставить другую колонку

## 5. Управление ссылками между таблицами

### Создать ссылочный термин
```bash
POST https://integram.io/{db}/_d_ref/{id_целевой_таблицы}?JSON=1
Body: _xsrf={xsrf}&token={token}
```
**Ответ:** `{ "obj": 650 }` ← ID ссылки

### Добавить ссылку как колонку в таблицу
```bash
POST https://integram.io/{db}/_d_req/{id_таблицы}?JSON=1
Body: t={id_ссылки}&_xsrf={xsrf}&token={token}
```
**Ответ:** `{ "id": 651 }` ← ID реквизита ссылки

### Включить/отключить множественный выбор у ссылки
```bash
POST https://integram.io/{db}/_d_multi/{id_реквизита_ссылки}?JSON=1
Body: _xsrf={xsrf}&token={token}
```

### Задать псевдоним для ссылочной колонки
```bash
POST https://integram.io/{db}/_d_alias/{id_реквизита_ссылки}?JSON=1
Body: val={псевдоним}&_xsrf={xsrf}&token={token}
```

### Получить значения справочника для ссылки
```bash
GET https://integram.io/{db}/_ref_reqs/{id_реквизита_ссылки}?JSON=1&id={id_записи}
Body: token={token}
```
**Ответ:** `{ "650": "Значение1", "660": "Значение2" }`

## 6. Управление записями (данными)

### Создать запись
```bash
POST https://integram.io/{db}/_m_new/{id_таблицы}?JSON=1
Body: up=1&t{id_реквизита1}={значение1}&t{id_реквизита2}={значение2}&_xsrf={xsrf}&token={token}
```
**Параметр up:** родитель (для корневых записей всегда 1)

**Ответ:** `{ "obj": 649 }` ← ID записи

### Получить содержимое таблицы (список записей)

#### Формат JSON=1 (полный)
```bash
GET https://integram.io/{db}/object/{id_таблицы}?JSON=1
Body: token={token}
```
**Ответ:**
```json
{
  "object": [
    { "id": 649, "up": 1, "val": "victor_g", "base": 3 },
    { "id": 650, "up": 1, "val": "anna_m", "base": 3 }
  ],
  "reqs": {
    "649": { "643": { "type": "Проект", "order": 1, "value": "Ромашка", "base": "SHORT" } },
    "650": { "643": { "type": "Проект", "order": 1, "value": "Лютик", "base": "SHORT" } }
  }
}
```
Где в `object` каждая запись содержит: `id` — ID записи, `up` — ID родителя, `val` — значение первой колонки, `base` — ID типа первой колонки.
В `reqs` — реквизиты каждой записи: ключ — ID реквизита, `value` — значение, `base` — базовый тип (`SHORT`, `NUMBER`, `DATE` и т.д.).

#### Формат JSON_DATA=1 (компактный)
```bash
GET https://integram.io/{db}/object/{id_таблицы}?JSON_DATA=1
Body: token={token}
```
**Ответ:**
```json
[
  {"i":649,"u":1,"o":1,"r":["victor_g","Ромашка"]},
  {"i":650,"u":1,"o":2,"r":["anna_m","650:Лютик"]}
]
```
Где:
- `i` — ID записи
- `u` — ID родителя (`up`)
- `o` — порядок (`ord`)
- `r` — массив значений: первое значение — первая колонка, далее — реквизиты в порядке их добавления. Для ссылочных реквизитов значение имеет формат `"id_целевой_записи:отображаемое_значение"` (например `"650:Лютик"`), для обычных — просто строковое значение.

### Обновить/изменить запись
```bash
POST https://integram.io/{db}/_m_save/{id_записи}?JSON=1
Body: up=1&t{id_записи}={значение первой колонки}&t{id_реквизита1}={значение реквизита1}&_xsrf={xsrf}&token={token}
```

### Удалить запись
```bash
POST https://integram.io/{db}/_m_del/{id_записи}?JSON=1
Body: _xsrf={xsrf}&token={token}
```

## 7. Управление значениями в записях через _m_set

`_m_set` применяется для установки значений **любых реквизитов** записи — как ссылочных, так и обычных (SHORT, NUMBER, DATE и т.д.).

**Ограничения:**
- `_m_set` **нельзя применять к первой колонке** таблицы. Для первой колонки используется `_m_save` (параметр `t{id_записи}=значение`).
- `_m_set` **распространяется не только на ссылочные реквизиты**, но и на обычные реквизиты (с базовыми типами).

### Установить значение реквизита (ссылочного или обычного)
```bash
POST https://integram.io/{db}/_m_set/{id_записи}?JSON=1
Body: t{id_реквизита}={значение}&_xsrf={xsrf}&token={token}
```
Для ссылочного реквизита `{значение}` — это ID целевой записи; для обычного — строковое или числовое значение.

**Ответ:** `{ "id": 652, "obj": 660 }`

### Удалить значение ссылки (для единственного выбора)
```bash
POST https://integram.io/{db}/_m_set/{id_записи}?JSON=1
Body: t{id_реквизита_ссылки}= &_xsrf={xsrf}&token={token}
```

### Удалить значение множественной ссылки
```bash
POST https://integram.io/{db}/_m_del/{id_значения_ссылки}?JSON=1
Body: _xsrf={xsrf}&token={token}
```

## 8. Работа с отчётами (запросами)

Отчёты поддерживают три формата ответа, задаваемых параметром в URL.

### Формат JSON=1 (полный, с метаданными колонок)
```bash
GET https://integram.io/{db}/report/{id_отчёта}?JSON=1
Body: token={token}
```
**Ответ:**
```json
{
  "header": "Название отчёта",
  "columns": [
    { "id": 643, "name": "Название проекта", "type": 3, "format": "SHORT" },
    { "id": 644, "name": "Бюджет", "type": 13, "format": "NUMBER", "granted": 1 },
    { "id": 651, "name": "Клиент", "type": 650, "format": "SHORT", "ref": 1, "orig": 641 }
  ],
  "data": ["Ромашка", "500000", "Иванов"]
}
```
Поле `granted: 1` присутствует, если есть право записи. Поля `ref: 1` и `orig` присутствуют для ссылочных колонок.
`data` — массив значений первой строки отчёта (одна строка).

### Формат JSON_KV=1 (массив объектов, все строки)
```bash
GET https://integram.io/{db}/report/{id_отчёта}?JSON_KV=1
Body: token={token}
```
**Ответ:** (файл `{название_отчёта}.json`)
```json
[
  { "Название проекта": "Ромашка", "Бюджет": "500000", "Клиент": "Иванов" },
  { "Название проекта": "Лютик",   "Бюджет": "300000", "Клиент": "Петров" }
]
```
Ключи объектов — названия колонок отчёта. Ответ возвращается как файл для скачивания.

### Формат JSON_DATA=1 (одна строка, ключ-значение)
```bash
GET https://integram.io/{db}/report/{id_отчёта}?JSON_DATA=1
Body: token={token}
```
**Ответ:**
```json
{ "Название проекта": "Ромашка", "Бюджет": "500000", "Клиент": "Иванов" }
```
Возвращает только первую строку результата в виде плоского объекта (ключ — название колонки).

### Выполнить отчёт с параметрами
```bash
POST https://integram.io/{db}/report/{id_отчёта}?JSON=1
Body: param1=value1&_xsrf={xsrf}&token={token}
```

## 10. Последовательность команд для создания структуры

### Шаг 1. Создать таблицу
```bash
POST .../_d_new?JSON=1  t=3&val=Проект
→ получаем obj=641
```

### Шаг 2. Создать колонку
```bash
POST .../_d_new?JSON=1  t=3&val=Название
→ получаем obj=642
```

### Шаг 3. Добавить колонку в таблицу
```bash
POST .../_d_req/641?JSON=1  t=642
→ получаем id=643 (ID реквизита)
```

### Шаг 4. Создать ссылку на другую таблицу
```bash
POST .../_d_ref/641?JSON=1
→ получаем obj=650 (ID ссылки)
```

### Шаг 5. Добавить ссылку как колонку
```bash
POST .../_d_req/644?JSON=1  t=650
→ получаем id=651 (ID реквизита ссылки)
```

### Шаг 6. Добавить запись
```bash
POST .../_m_new/641?JSON=1  up=1&t643=Ромашка
→ получаем obj=660 (ID записи)
```

### Шаг 7. Установить ссылку
```bash
POST .../_m_set/660?JSON=1  t651=649
```

## 11. Маппинг: ID → что это

| Где получен | Что это | Где используется |
|-------------|---------|------------------|
| `_d_new` → `obj` | ID таблицы | `_d_req/{id}`, `_m_new/{id}`, `metadata/{id}` |
| `_d_new` → `obj` (для колонки) | ID колонки | `_d_req` → параметр `t` |
| `_d_req` → `id` | ID реквизита | `_m_new` → `t{id}`, `_m_save` → `t{id}` |
| `_d_ref` → `obj` | ID ссылки | `_d_req` → параметр `t` |
| `_m_new` → `obj` | ID записи | `_m_set/{id}`, `_m_save/{id}`, `_m_del/{id}` |
| `_d_req` (для ссылки) → `id` | ID реквизита ссылки | `_m_set` → `t{id}` |

## 12. Коды ошибок

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `No authorization token provided` | Нет `token` в теле | Добавить `--form 'token="..."'` |
| `Invalid _xsrf token` | Неверный `_xsrf` | Использовать `_xsrf` из ответа `auth` |
| `Table not found` | Неверный ID таблицы | Проверить ID из `_d_new` |
| `Column not found` | Использован ID колонки вместо ID реквизита | Использовать `id` из `_d_req` |
