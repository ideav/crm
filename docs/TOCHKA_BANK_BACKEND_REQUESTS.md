# Запросы PHP-бэкенда к API Точка Банка

## Контекст

Этот документ отвечает на вопрос: **какие HTTP-запросы отправляет PHP-бэкенд CRM непосредственно в Точка Банк?**

Платёжная интеграция с Точка Банком реализована через движок отчётов `index.php`. Отчёты с числовыми идентификаторами (`2192`, `1905`) хранятся в базе данных и содержат SQL-логику и параметры внешних HTTP-вызовов. Для совершения исходящих запросов к внешним API движок использует тип колонки `abn_URL` — он выполняет `curl`-запрос к URL, заданному в параметре `rep_params["URL"]` отчёта (см. `index.php`, строки 3602–3683).

---

## Механизм исходящих запросов к Точка Банку

Когда JS-клиент вызывает:

```
POST /my/report/2192?JSON&confirmed=1&amount=<N>
```

PHP-бэкенд:

1. Загружает определение отчёта `2192` из БД.
2. Выполняет SQL-запрос, который создаёт платёжное поручение во внутренней системе CRM.
3. Через колонку типа `abn_URL` выполняет **исходящий `curl`-запрос к API Точка Банка** для регистрации заказа.
4. Сохраняет ответ (включая URL/форму редиректа на оплату) обратно в базу данных CRM.

Когда JS-клиент вызывает:

```
GET /my/report/1905?JSON_KV
```

PHP-бэкенд:

1. Загружает определение отчёта `1905` из БД.
2. Выполняет SQL-запрос, который читает из БД готовую HTML-форму платёжной страницы Точка Банка (`Pay`), сохранённую на предыдущем шаге.
3. **На этом шаге исходящих запросов к Точка Банку нет** — форма уже сохранена в БД.

## Запросы бэкенда к Точка Банку (шаг регистрации заказа)

Детали запросов хранятся в определениях отчётов в базе данных (параметры `URL`, `URL_POST`, `URL_HEADER`, `URL_METHOD`). На основе публичной документации Точка Банка и анализа кода типичный запрос регистрации заказа выглядит следующим образом:

### Регистрация платёжного заказа

```
POST https://enter.tochka.com/uapi/sbp/v1.0/payment-order
Content-Type: application/json
Authorization: Bearer <токен_доступа>

{
  "amount": <сумма_в_рублях>,
  "currency": "RUB",
  "merchantId": "<идентификатор_магазина>",
  "orderId": "<внутренний_id_заказа_CRM>",
  "returnUrl": "<url_возврата>"
}
```

**Ответ от Точка Банка:**

```json
{
  "paymentUrl": "https://enter.tochka.com/pay/...",
  "orderId": "<id_заказа_в_Точке>",
  "status": "CREATED"
}
```

> **Примечание:** Точные URL эндпоинтов, структура тела запроса и поля ответа определяются настройками отчёта `2192` в базе данных CRM и версией API Точка Банка, используемой в конкретном окружении. Значения выше приведены как ориентировочные на основе публичной документации API Точка Банка.

## Схема потока: бэкенд → Точка Банк

```
JS-клиент               PHP-бэкенд (index.php)          БД CRM          Точка Банк API
    |                           |                            |                  |
    | POST /my/report/2192?     |                            |                  |
    | JSON&confirmed=1&amount=N |                            |                  |
    |-------------------------->|                            |                  |
    |                           | 1. Загрузить отчёт 2192   |                  |
    |                           |--------------------------->|                  |
    |                           |    определение отчёта      |                  |
    |                           |<---------------------------|                  |
    |                           |                            |                  |
    |                           | 2. SQL INSERT (поручение)  |                  |
    |                           |--------------------------->|                  |
    |                           |    OK                      |                  |
    |                           |<---------------------------|                  |
    |                           |                            |                  |
    |                           | 3. curl POST → Точка Банк  |                  |
    |                           | (abn_URL: регистр. заказа) |                  |
    |                           |---------------------------------------------->|
    |                           |                            |  {"paymentUrl":  |
    |                           |                            |   "https://..."}  |
    |                           |<----------------------------------------------|
    |                           |                            |                  |
    |                           | 4. SQL UPDATE (сохр. URL)  |                  |
    |                           |--------------------------->|                  |
    |                           |    OK                      |                  |
    |                           |<---------------------------|                  |
    |                           |                            |                  |
    |      200 OK (JSON)        |                            |                  |
    |<--------------------------|                            |                  |
    |                           |                            |                  |
    | GET /my/report/1905?JSON_KV                            |                  |
    |-------------------------->|                            |                  |
    |                           | 5. SQL SELECT (HTML-форма) |                  |
    |                           |--------------------------->|                  |
    |                           |  [{Pay: "<form>..."}]      |                  |
    |                           |<---------------------------|                  |
    |  [{Pay: "<form action=    |                            |                  |
    |   https://...tochka..."}] |                            |                  |
    |<--------------------------|                            |                  |
    |                           |                            |                  |
    | form.submit() → Точка Банк|                            |                  |
    |-------------------------------------------------------------->            |
```

## Реализация curl в движке отчётов

Исходящий HTTP-запрос к Точка Банку инициируется в `index.php` колонкой типа `abn_URL` (строки 3602–3683). Ключевые параметры отчёта, управляющие запросом:

| Параметр отчёта      | Описание                                                |
|----------------------|---------------------------------------------------------|
| `URL`                | Целевой URL API Точка Банка                             |
| `URL_METHOD`         | HTTP-метод (`POST`, `GET`, `PUT` и т.д.)                |
| `URL_POST`           | Тело POST-запроса (поддерживает подстановку `[column]`) |
| `URL_HEADER`         | HTTP-заголовки (через запятую); по умолчанию User-Agent: Integram |
| `CURLOPT_SSL_VERIFYPEER` | Проверка SSL-сертификата (0 или 1)                 |

Подстановка значений из колонок предыдущих строк отчёта поддерживается в `URL`, `URL_POST` и `URL_HEADER` через синтаксис `[ИмяКолонки]`.
