# Псевдоним таблицы (table alias) — issue #2967

## Задача

В Интеграме таблица называется так же, как её первая колонка: первая колонка
таблицы — это значение самого термина (`$z.val`), отдельной строки под неё нет.
Пользователи хотят давать таблице произвольное имя, не меняя имя первой колонки.

Решение: у таблицы появляется **псевдоним** — отображаемое имя, которое хранится
в атрибутах первой колонки и переопределяет «сырое» имя везде, где раньше
показывалось имя первой колонки. Само имя первой колонки (и, значит, имя термина)
остаётся прежним, поэтому ссылки, поиск по имени и совместимость не ломаются.

## Где хранится

У первой колонки нет собственной строки в `$z`, поэтому её атрибуты негде
держать. Для этого вводится **строка-самоописание** (self-descriptor requisite):

```
(up = tableId, t = tableId, val = attrsJson, ord = 0)
```

Строка ссылается сама на свою таблицу (`req.t == tableId`), что делает её
однозначным маркером. В `attrsJson` лежит обычный JSON атрибутов поля
(`include/field_attrs.php`), а псевдоним — в ключе `alias`. Такая же схема
атрибутов уже используется для остальных колонок (issue #2485), так что
формат единый.

Реконструкция структуры (`metadata`, `terms`) обязана:

1. **не** выдавать строку-самоописание как колонку;
2. отдавать её атрибуты как `attrs`/`alias` на уровне таблицы;
3. оставлять таблицу видимой, даже если у неё нет других колонок;
4. не портить поле `referenced` у таблицы, на которую ссылаются.

Эти четыре свойства проверяются в `experiments/test-issue-2967-table-alias.php`.

## API

### Чтение

- `GET /{db}/metadata` и `GET /{db}/metadata/{id}` — у таблицы появляются поля
  `attrs` (сырой JSON) и `alias` (распакованный псевдоним), если он задан.
  Строка-самоописание не присутствует в `reqs`.
- `GET /{db}/terms?JSON` — у элемента списка появляется поле `alias`, если задан.
  Поле `name` по-прежнему содержит «сырое» имя таблицы.

### Запись

- `POST /{db}/_t_alias/{tableId}?JSON` с `val=<псевдоним>` (и `_xsrf`) —
  устанавливает псевдоним. Пустой `val` очищает его. Алиасы синонима:
  `_settablealias`. Требуется право `WRITE` на структуру таблицы
  (`Grant_1level`). Символ `:` в псевдониме запрещён (как и для алиасов колонок).
  Эндпоинт ищет строку-самоописание и обновляет её `val`, либо создаёт её.

### Прочие точки вывода имени таблицы (на будущее)

Эти места по-прежнему показывают «сырое» имя термина. Менять их в рамках issue
не требовалось (issue ограничил объём workspace `tables.html`, `table.html` и
компонентом `js/integram-table.js`), но они перечислены для полноты:

- Шаблонизатор `&Settings` и серверные HTML-страницы старого образца
  (`object.html`, `edit_types.html` и т.п.), которые рендерят имя термина
  напрямую из БД, минуя `metadata`.
- Хлебные крошки/заголовки старого UI, читающие `$z.val` напрямую.
- Экспорт (`b3x`, Google Sheets sync) — используют машинные имена колонок;
  псевдоним там сознательно не применяется.
- Ссылочные колонки (справочники) в других таблицах показывают значения строк,
  а не имя таблицы, поэтому псевдонимом не затрагиваются.

## Поведение

- Псевдоним **переопределяет** показ; «сырое» имя первой колонки и термина не
  меняется, поэтому поиск по имени, ссылки и совместимость сохраняются.
- В настройках колонок первая колонка показывается как `Псевдоним (Имя)`, по
  аналогии с псевдонимами ссылочных колонок.
- Поиск таблиц в workspace `tables.html` находит таблицу и по «сырому» имени, и
  по псевдониму.
