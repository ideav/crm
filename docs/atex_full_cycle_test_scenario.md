# Полный цикл тестирования atex

**Задача:** [#2978](https://github.com/ideav/crm/issues/2978)
**Цель:** дать тестировщику один воспроизводимый сценарий, который проходит через
актуальные шаблоны `templates/atex/` от заказа до отгрузки и проверки клиентского
портала.

Документ рассчитан на базу `atex`, где уже выполнены:

- структура из [`docs/atex_metadata.json`](atex_metadata.json);
- роли и пользователи из [`docs/atex_roles_users.json`](atex_roles_users.json);
- шаблоны из `templates/atex/` и ассеты из `download/atex/`;
- пункты меню из [`docs/atex_menu.json`](atex_menu.json), созданные
  [`docs/create_atex_menu.ps1`](create_atex_menu.ps1).

## Тестовые учётные записи

Используйте готовых пользователей из `docs/atex_roles_users.json`:

| Роль | Логин / пароль | Рабочие места |
|---|---|---|
| Администратор | `admin / admin_atex_2026` | штатные таблицы для подготовки справочников |
| Менеджер | `manager / manager_atex_2026` | `templates/atex/orders.html` |
| Диспетчер | `dispatcher / dispatcher_atex_2026` | `templates/atex/cut-calc.html`, `templates/atex/production-planning.html` |
| Оператор | `operator / operator_atex_2026` | `templates/atex/slitter.html`, `templates/atex/sleeve-cutter.html`, `templates/atex/cut-map.html` |
| Кладовщик | `operator / operator_atex_2026` | `templates/atex/intake.html`, `templates/atex/warehouse.html` |
| Руководитель | `director / director_atex_2026` | `templates/atex/dashboards.html` |
| Клиент | `client / client_atex_2026` | `templates/atex/portal.html` |

## Копипаст-набор

### Клиент

Создать в таблице `Клиент` через штатный UI Интеграма до начала сценария.
Логин `client` нужен, чтобы клиентский портал сопоставил пользователя с записью
клиента.

```text
Клиент: ООО "Ромашка-Термолента"
Логин: client
Пароль: client_atex_2026
Email: qa-romashka@example.test
Телефон: +7 495 010-29-78
```

### Производственные справочники

Создать через штатный UI Интеграма до начала сценария. Значения слева от двоеточия
соответствуют названию записи или поля.

```text

Полное название: Термобумага Jumbo Thermal, ширина 910 мм, намотка 4000 м
Ширина, мм: 910
Длина рулона, м: 4000
Примечания: тестовый рулон для полного цикла #2978

Слиттер: SL-01
Статус: Доступен
Примечания: основной тестовый слиттер

Втулкорез: TC-76
Диаметр, мм: 76
Статус: Доступен
```

## Выбор из списков

Если поле открывается как список, выбирайте такие значения:

| Поле | Значение |
|---|---|
| Клиент | `ООО "Ромашка-Термолента"` |
| Вид сырья | `Jumbo Thermal 910/4000` |
| Тип резки | `ТТ-58 ECO 57x40` |
| Слиттер | `SL-01` |
| Втулкорез | `TC-76` |
| Статусы заказа | `Новый` → `Согласован` → `В производстве` → `Выполнен` |
| Статусы позиции | `Новая` → `В работе` → `Готова` → `Отгружена` |
| Статусы планирования резки | `Запланирована` → `В очереди` |
| Статусы пульта слиттера | `Ожидает` → `Наладка` → `В работе` → `Завершён` |
| Статусы втулкореза | `Ожидает` → `В работе` → `Готово` |
| Статусы партии ГП | `Есть` → `Зарезервирован` → `Отгружен` |
| Статусы обеспечения | `Зарезервировано` → `Выполнено` |

## Сценарий полного цикла

### 1. Подготовить справочники

1. Войти администратором или пользователем с доступом к штатным таблицам.
2. Создать клиента, вид сырья, слиттер и втулкорез из раздела
   [Копипаст-набор](#копипаст-набор).
3. Проверить, что запись клиента содержит `Логин: client`.

### 2. Принять сырьё на склад

1. Войти пользователем склада.
2. Открыть `templates/atex/intake.html`.
3. Создать партию сырья `RM-2026-05-31-01`.
4. Выбрать `Jumbo Thermal 910/4000`, указать дату `2026-05-31` и
   `Получено, м²: 3640`.
5. Проверить, что `Остаток, м²` равен `3640`, а партия есть в FIFO-списке.

### 3. Создать тип резки

1. Войти как `dispatcher / dispatcher_atex_2026`.
2. Открыть `templates/atex/cut-calc.html`.
3. Создать тип резки `ТТ-58 ECO 57x40`.
4. Добавить две полосы из раздела [Тип резки](#тип-резки).
5. Проверить расчёт: `Итого ножей = 18`, `Остаток, мм = 20`.

### 4. Создать заказ

1. Войти как `manager / manager_atex_2026`.
2. Открыть `templates/atex/orders.html`.
3. Нажать `Создать заказ` и заполнить заказ с меткой `АТХ-2026-0001` в
   примечании. Номер заказа в списке присваивается сервером; запишите его для
   следующих шагов.
4. Добавить позицию заказа `ТТ-58 ECO 57x40 / 57 мм / 1200 м`.
5. Сменить статус заказа на `Согласован`, статус позиции на `В работе`.
6. Обновить список и проверить, что заказ виден с одной позицией.

### 5. Запланировать производство

1. Войти как `dispatcher / dispatcher_atex_2026`.
2. Открыть `templates/atex/production-planning.html`.
3. Создать производственную резку: `SL-01`, `ТТ-58 ECO 57x40`,
   `RM-2026-05-31-01`, `Дата план: 2026-06-01`.
4. Перевести резку в `В очереди`.
5. В блоке обеспечения выбрать позицию заказа, созданную на шаге 4, указать
   `Метраж, м: 1200` и статус `Зарезервировано`.
6. Проверить, что резка появилась в очереди по слиттеру `SL-01`.
7. Вернуться в `templates/atex/orders.html` и перевести заказ в статус
   `В производстве`.

### 6. Проверить карту раскроя

1. Войти как `operator / operator_atex_2026`.
2. Открыть `templates/atex/cut-map.html`.
3. Выбрать производственную резку, созданную для сценария `АТХ-2026-0001`.
4. Проверить, что карта показывает 18 ножей, занято `890 мм`, остаток `20 мм`,
   переполнения нет.

### 7. Выполнить резку на слиттере

1. Открыть `templates/atex/slitter.html`.
2. Выбрать резку по слиттеру `SL-01`.
3. Если резка пришла из планирования в статусе `В очереди`, вручную выбрать
   первый шаг `Ожидает`, затем последовательно перевести статусы
   `Ожидает` → `Наладка` → `В работе`.
4. Ввести `Счётчик нач.: 1000`, `Счётчик кон.: 2200`, `Брак, м²: 12`.
5. Добавить расход сырья по партии `RM-2026-05-31-01` на `1200`.
6. Перевести резку в `Завершён`.
7. Проверить, что остаток партии сырья уменьшился, а событие смены создано.

### 8. Выполнить задание на втулки

1. Открыть `templates/atex/sleeve-cutter.html`.
2. Выбрать задание на резку `SL-01 / ТТ-58 ECO 57x40`, созданную для сценария
   `АТХ-2026-0001`.
3. Указать `TC-76`, `Кол-во план: 10`, `Диаметр, мм: 76`.
4. Перевести статус `Ожидает` → `В работе` → `Готово`.
5. Указать `Кол-во факт: 10` и проверить, что сводка показывает выполнение
   `100%`.

### 9. Принять готовую продукцию и отгрузить

1. Войти пользователем склада.
2. Открыть `templates/atex/warehouse.html`.
3. Создать партию ГП из завершённой резки.
4. Заполнить: `Ширина, мм: 57`, `Кол-во рулонов: 10`, `Метраж, м: 1200`,
   `Адрес хранения: A-01-03`, `Статус: Есть`.
5. Выполнить FIFO-привязку партии ГП к обеспечению позиции заказа.
6. Перевести партию ГП `Есть` → `Зарезервирован` → `Отгружен`.
7. Перевести обеспечение в `Выполнено`, позицию заказа в `Отгружена`, заказ в
   `Выполнен`.

### 10. Проверить руководительский дашборд

1. Войти как `director / director_atex_2026`.
2. Открыть `templates/atex/dashboards.html`.
3. Проверить, что в сводках есть заказ в статусе `Выполнен`, резка по `SL-01`,
   выпуск ГП и уменьшенный остаток сырья.

### 11. Проверить клиентский портал

1. Войти как `client / client_atex_2026`.
2. Открыть `templates/atex/portal.html`.
3. Проверить, что виден только заказ клиента `ООО "Ромашка-Термолента"`.
4. Раскрыть заказ с меткой `АТХ-2026-0001` и проверить позицию:
   `ТТ-58 ECO 57x40 / 57 мм / 1200 м`, статус `Отгружена`.
5. Убедиться, что чужие заказы, если они есть в базе, не показаны.

## Критерии приёмки

- Все 10 актуальных рабочих мест из `templates/atex/` пройдены хотя бы один раз.
- Тестировщик не придумывает данные вручную: все значения берутся из
  копипаст-набора или из раздела выбора.
- Заказ с меткой `АТХ-2026-0001` прошёл путь `Новый` → `Согласован` →
  `В производстве` → `Выполнен`.
- Производственная резка прошла путь планирования и слиттера до `Завершён`.
- Партия сырья была принята и частично списана.
- Партия ГП была принята, зарезервирована и отгружена.
- Дашборд руководителя отражает результат полного цикла.
- Клиент `client` видит свой заказ в портале и не видит чужие заказы.
