# Инструкция по проектированию структур данных в Интеграме

## Общие принципы

### Первая колонка таблицы — это имя (идентификатор) самого объекта

**Важное правило:** первая колонка любой таблицы должна быть **собственным идентификатором объекта**, а не ссылкой на другой объект.

| Правильно | Неправильно |
|-----------|-------------|
| Таблица «Стратегия»: первая колонка — «Версия стратегии» или «Код стратегии» | Таблица «Стратегия»: первая колонка — ссылка на «Проект» |
| Таблица «Платёж»: первая колонка — «Номер платежа» | Таблица «Платёж»: первая колонка — ссылка на «Проект» |
| Таблица «Задача»: первая колонка — «Название задачи» | Таблица «Задача»: первая колонка — ссылка на «Проект» |

### Почему это важно

1. **У каждой сущности есть своё «имя»** — номер, код, версия, идентификатор
2. **Ссылки на другие объекты** — это отдельные колонки (например, «Проект», «Ответственный», «Исполнитель»)
3. **Пользователям понятны имена объектов**, а не внутренние ID

### Внутренние ID

В Интеграме у каждой записи есть внутренний числовой ID (поле `i` в ответе API). Он используется для технических целей (ссылок, связей), но **пользователи его не видят** и не работают с ним. Пользователи работают с первой колонкой — понятным им идентификатором.

## Типы данных в Интеграме

| Код | Тип | Описание | Когда использовать |
|-----|-----|----------|---------------------|
| 3 | SHORT | Короткая строка (до 127 символов) | Имена, коды, номера телефонов |
| 8 | CHARS | Длинная строка | Email, URL, адреса |
| 9 | DATE | Дата | ДД.ММ.ГГГГ |
| 4 | DATETIME | Дата и время | ДД.ММ.ГГГГ ЧЧ:ММ |
| 13 | NUMBER | Целое число | Суммы, количество, бюджет |
| 14 | SIGNED | Число с десятичной частью | Проценты, коэффициенты |
| 11 | BOOLEAN | Да / Нет (1 или 0) | Флаги, признак активности |
| 12 | MEMO | Многострочный текст | Описания, комментарии, транскрипции |
| 10 | FILE | Файл | Изображения, аудио, вложения |
| 6 | PWD | Пароль (маскируется) | Токены, пароли |

## Системные таблицы (уже существуют в каждой базе)

Эти таблицы создаются автоматически при создании базы. Их не нужно создавать заново.

### Таблица «Пользователь»

| Поле | Тип | Примечание |
|------|-----|------------|
| **Логин** | SHORT | **Первая колонка** — уникальный идентификатор пользователя |
| Роль | SHORT (ссылка на «Роль») | Роль доступа |
| Email | CHARS | Электронная почта |
| Телефон | SHORT | Номер телефона |
| Имя | SHORT | Полное имя |
| Фото | FILE | Аватар |
| Дата | DATE | Дата регистрации |
| Активен | BOOLEAN | Работает/заблокирован |

**Пример данных:**

| Логин | Роль | Email | Телефон | Имя | Дата | Активен |
|-------|------|-------|---------|-----|------|---------|
| ivanov | admin | ivanov@company.ru | +7(999)111-22-33 | Иван Иванов | 01.01.2025 | Да |
| petrova | user | petrova@company.ru | +7(999)444-55-66 | Анна Петрова | 15.02.2025 | Да |

### Таблица «Роль»

| Поле | Тип | Примечание |
|------|-----|------------|
| **Название роли** | SHORT | **Первая колонка** — admin, user, guest, founder, team |
| Описание | MEMO | Пояснение |

**Пример данных:**

| Название роли | Описание |
|---------------|----------|
| admin | Полный доступ ко всем объектам |
| user | Обычный пользователь |
| founder | Сооснователь, полный доступ |
| team | Маркетолог, ограниченный доступ |

## Шаблоны для создания таблиц

### Шаблон 1: Таблица-справочник (независимые объекты)

| Поле | Тип | Примечание |
|------|-----|------------|
| **Название** | SHORT | **Первая колонка** — уникальное имя |
| Описание | MEMO | Пояснение |
| Активен | BOOLEAN | Да / Нет |

**Примеры:** Статусы, Категории, Типы, Теги

### Шаблон 2: Таблица-документ (события, операции)

| Поле | Тип | Примечание |
|------|-----|------------|
| **Номер документа** | SHORT | **Первая колонка** — уникальный номер |
| Дата | DATE | Дата документа |
| Ссылка на основной объект | SHORT (ссылка) | Например, Проект, Клиент |
| Сумма / Количество | NUMBER | Основной показатель |
| Статус | SHORT | Состояние документа |

**Примеры:** Платежи, Заказы, Акты, Счета

### Шаблон 3: Таблица-задача (рабочие единицы)

| Поле | Тип | Примечание |
|------|-----|------------|
| **Название задачи** | MEMO | **Первая колонка** — описание |
| Проект | SHORT (ссылка) | К какому проекту |
| Исполнитель | SHORT (ссылка на «Пользователь») | Кто делает |
| Дедлайн | DATE | Срок |
| Статус | SHORT | open / done |

**Примеры:** Задачи, Поручения, Инциденты

### Шаблон 4: Таблица-версия (для документов с историей)

| Поле | Тип | Примечание |
|------|-----|------------|
| **Версия** | SHORT | **Первая колонка** — v1.0, v2.0, draft |
| Основной объект | SHORT (ссылка) | Например, Проект, Документ |
| Ссылка на файл | CHARS | URL |
| Статус версии | SHORT | draft / approved |
| Дата создания | DATE | Когда создана |

**Примеры:** Стратегии, Договоры, Спецификации

## Частые ошибки и их исправление

| Ошибка | Почему неправильно | Как исправить |
|--------|-------------------|---------------|
| Первая колонка — ссылка на проект | У стратегии нет собственного имени | Сделать первой колонкой «Версия стратегии» |
| Первая колонка — ссылка на клиента | У платежа нет своего номера | Сделать первой колонкой «Номер платежа» |
| Первая колонка — дата | Дата сама по себе не уникальна | Добавить составной идентификатор («Проект + Дата») |
| Первая колонка — булево значение | Не может быть идентификатором | Добавить текстовое имя |

## Создание ссылок между таблицами

Когда нужно связать таблицы, процесс такой:

1. **Первая таблица** (например, «Проект») имеет свою первую колонку («Название проекта»)
2. **Вторая таблица** (например, «Стратегия») имеет свою первую колонку («Версия стратегии»)
3. Во второй таблице создаётся колонка типа SHORT, которая является **ссылкой** на первую таблицу
4. В этой колонке хранится значение из первой колонки первой таблицы («Название проекта»)

**Пример связи:**

Таблица «Проект»:
| Название проекта |
|------------------|
| Альфа |
| Бета |

Таблица «Стратегия»:
| Версия стратегии | Проект (ссылка) |
|------------------|-----------------|
| v1.0 | Альфа |
| v2.0 | Альфа |
| v1.0 | Бета |

## Итог: золотые правила

1. **Первая колонка** — собственное имя объекта (не ссылка!)
2. **Имена на русском** — понятны пользователю
3. **Единственное число** — «Пользователь», а не «Пользователи»
4. **Ссылки** — в отдельных колонках, не в первой
5. **Внутренние ID** — есть, но пользователь их не видит
