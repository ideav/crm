# Правила UI/UX (рабочие места ideav/crm, atex)

Накопительный свод UI/UX-правил для рабочих мест. Дополняй при каждом новом
решении или фидбеке от пользователя. Дополняет
[`WORKSPACE_DEVELOPMENT_GUIDE.md`](WORKSPACE_DEVELOPMENT_GUIDE.md) (там — техника,
здесь — поведение интерфейса для пользователя).

## Таблицы и поячейковая правка

- **Верстка не «прыгает» при входе ячейки в режим правки.** Высота строки и
  ширина колонок не меняются между отображением и правкой. Практика:
  - фиксировать высоту редактируемых ячеек (`td.<cell> { height: … }`);
  - НЕ менять `padding` ячейки между отображением и правкой;
  - контрол правки — `box-sizing: border-box; width: 100%`, без `min-width`,
    расширяющего колонку;
  - индикатор активной ячейки — через `box-shadow: inset …` (не меняет геометрию),
    не через рамку/padding, добавляющие размер.
- **Поле ввода вписывается ровно в отведённое место ячейки** — без переполнения по
  ширине и высоте.
- **Выпадающие списки** (ref-select, автокомплит) позиционируются абсолютно
  (`position: absolute`), чтобы не сдвигать строку/таблицу.

*Реализовано в РМ `orders`: `download/atex/css/orders.css` —
`.atex-orders-cell`, `.atex-orders-cell-input`, `.atex-orders-cell .atex-orders-ref-select`
(2026-06-02).*
