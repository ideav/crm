# issue #4327 — точечный фикс для старой (VERSION 8) базы `upsound`

**Симптом.** При импорте plain-данных запись, встреченная в файле дважды, двоила
плоские реквизиты: у объекта появлялись несколько строк одного реквизита с одним
значением (в тикете — реквизит 293 `'4660275212865'` по 4 раза на объект).

**Причина.** Объект создаётся сразу через `Insert()` (коммит в БД), а его реквизиты
копятся в `Insert_batch` и до закрытия батча в БД не попадают. На повторной строке
файла `SELECT` находит объект по уникальности, но его реквизитов ещё не видит
(`$reqs` пуст) → ветка обновления не срабатывает → второй `Insert_batch` → дубль
плоского реквизита. У старой версии `Insert_batch` вообще без дедупликации.

**В основном коде (`index.php` на `main`, VERSION 118) это уже закрыто** — PR #4146 /
issue #4140 (влит 2026-07-11): батч переписан на массив `[up][вид][колонка][слот]`,
скалярный реквизит перезаписывает застрявшую строку вместо дописывания. Обновить прод
`upsound` целиком нельзя — там VERSION 8, несовместимая с текущим `index.php`.

## Что здесь

Точечный патч в **ту самую VERSION-8** версию (`experiments/index_old.php`,
байт-в-байт равна деплою `index_upsound_260723.php`). Меняется только функция
`Insert_batch`: добавлен дедуп точных кортежей `(up,ord,t,val)` в пределах батча
(механизм из #2785), усиленный тем, что индекс `SQLbatchSeen` держится **до закрытия
батча** и не сбрасывается на промежуточном 31 000-символьном флаше — иначе дубли,
разнесённые флашем в большом импорте, проскакивают. Заодно пустой «Close batch»
больше не роняет в батч мусорную строку `(,,,'')`.

| файл | что это |
|------|---------|
| `index_upsound_4327.patch` | минимальный unified-diff (единственный хунк в `Insert_batch`) |
| `index_upsound_260723.fixed.php` | готовый к деплою файл (VERSION 8, CRLF/UTF-8 как оригинал) — переименовать в `index.php` |
| `index_upsound_4327_batch_dedup.test.php` | самодостаточный харнесс (8 проверок), `php index_upsound_4327_batch_dedup.test.php` → `ALL TESTS PASSED` |

## Деплой

Заменить на инстансе `upsound` только `index.php` содержимым
`index_upsound_260723.fixed.php`. `VERSION` не трогается (остаётся 8), ассеты не задеты.
