# Integram Table Component

Компонент для отображения табличных данных из Integram API с полной поддержкой пагинации, фильтрации и настройки колонок.

## Возможности

- ✅ **Бесконечный скролл** - автоматическая подгрузка данных при прокрутке
- ✅ **Умный счетчик записей** - показывает загруженные записи, общее количество по клику
- ✅ **Динамические колонки** - автоматическая адаптация к структуре данных
- ✅ **Форматы данных** - поддержка SHORT, CHARS, NUMBER, DATE, DATETIME, BOOLEAN, MEMO, HTML, BUTTON, PWD, PATH
- ✅ **Drag & Drop** - перетаскивание колонок для изменения порядка
- ✅ **Изменение ширины колонок** - ручная настройка ширины колонок с сохранением в cookies
- ✅ **Настройки колонок** - скрытие/показ колонок через модальное окно
- ✅ **Настройки таблицы** - компактный/просторный режим, размер страницы, сокращение длинных значений, скрытие подписей кнопок меню
- ✅ **Фильтрация** - 13 типов фильтров для разных типов данных
- ✅ **Автоматическое применение фильтров** - фильтры применяются при вводе текста (для Пусто/Не пусто - мгновенно)
- ✅ **Очистка фильтров** - кнопка очистки всех фильтров появляется при наличии активных фильтров (включая Пусто/Не пусто)
- ✅ **Сортировка** - сортировка по любому полю (по возрастанию/убыванию) кликом на заголовок колонки
- ✅ **Группировка** - группировка таблицы по одному или нескольким полям с объединением ячеек
- ✅ **Липкий заголовок** - шапка таблицы и строка фильтров остаются видимыми при прокрутке
- ✅ **Вертикальные разделители** - тонкие линии между колонками для улучшенной читаемости
- ✅ **Скрытие служебных колонок** - автоматическое скрытие колонок с суффиксами ID и Стиль
- ✅ **Динамическое стилизование** - применение стилей к ячейкам через колонки Стиль (сохраняется при загрузке из cookies)
- ✅ **Редактирование записей** - модальные формы для редактирования существующих записей
- ✅ **Редактирование в ячейках** - inline-редактирование значений прямо в таблице с навигацией клавиатурой
- ✅ **Подчиненные таблицы вкладками** - отображение связанных таблиц в отдельных вкладках с возможностью редактирования
- ✅ **Многоуровневые модальные окна** - корректное наложение форм редактирования друг на друга
- ✅ **Автоматическое расширение модальных окон** - расширение до 93% экрана для больших подчиненных таблиц
- ✅ **Поиск в справочниках** - живой поиск в выпадающих списках формы редактирования
- ✅ **Настройка видимости полей** - скрытие/показ полей в формах редактирования
- ✅ **Умные значения по умолчанию** - автозаполнение дат текущим значением, пропуск пустых полей для серверных defaults
- ✅ **Просмотр полных значений** - модальное окно для просмотра сокращенных значений
- ✅ **Липкий скроллбар** - нижний горизонтальный скроллбар для широких таблиц
- ✅ **Persistence** - сохранение настроек в cookies браузера
- ✅ **Адаптивность** - корректное отображение на разных устройствах
- ✅ **Выбор строк чекбоксами** - выделение записей для массовых операций (удаление)
- ✅ **Экспорт в Excel/CSV** - выгрузка данных таблицы в форматы XLSX, XLS, CSV
- ✅ **Поделиться ссылкой** - копирование URL с текущими фильтрами, группировкой и сортировкой

- ✅ **Уникальные значения колонки** - установка флага уникальности при редактировании колонки
- ✅ **Переименование колонок** - изменение имени колонки через форму редактирования
- ✅ **Дата-пикер в фильтрах** - нативные HTML5 date/datetime пикеры для DATE и DATETIME фильтров
- ✅ **Фиксированный счетчик записей** - счетчик остается на месте при горизонтальном скролле таблицы

## Быстрый старт

### 1. Подключение (рекомендуемый способ - data-атрибуты)

Просто добавьте атрибут `data-integram-table` к контейнеру и укажите параметры через data-атрибуты:

```html
<!DOCTYPE html>
<html>
<head>
    <link rel="stylesheet" href="/css/bootstrap.min.css">
    <link rel="stylesheet" href="/css/integram-table.css">
</head>
<body>
    <div id="my-table"
         data-integram-table
         data-api-url="/api/tasks"
         data-page-size="20"
         data-cookie-prefix="my-table"
         data-title="Мои задачи"
         data-instance-name="myTable"></div>

    <script src="/js/integram-table.js"></script>
    <!-- Таблица инициализируется автоматически! -->
</body>
</html>
```

**Преимущества:**
- ✅ Конфигурация в одном месте (в HTML)
- ✅ Не нужно писать JavaScript код
- ✅ Автоматическая инициализация при загрузке страницы
- ✅ Легко редактировать параметры

### 1б. Подключение (программный способ)

Если нужен контроль через JavaScript:

```html
<!DOCTYPE html>
<html>
<head>
    <link rel="stylesheet" href="/css/bootstrap.min.css">
    <link rel="stylesheet" href="/css/integram-table.css">
</head>
<body>
    <div id="my-table"></div>

    <script src="/js/integram-table.js"></script>
    <script>
        const myTable = new IntegramTable('my-table', {
            apiUrl: '/api/tasks',
            pageSize: 20,
            cookiePrefix: 'my-table',
            instanceName: 'myTable'
        });
    </script>
</body>
</html>
```

**Примечание:** Теперь компонент - это standalone JS модуль, не требующий jQuery!

```json
{

      "format": "CHARS",
      "name": "Задача",
      "granted": 1,
      "ref": 0
    }
  ],
  "data": [
    ["Значение 1"],
    ["Значение 2"]
  ],

```

### 3. Параметры конфигурации

```javascript
new IntegramTable('container-id', {
    // Общие параметры
    pageSize: 20,                  // Количество записей на странице
    cookiePrefix: 'table-name',    // Префикс для cookies
    title: 'Название таблицы',     // Опциональный заголовок
    instanceName: 'myTable',       // Имя экземпляра для window (обязательно!)
    onCellClick: (row, col, val) => {},  // Обработчик клика по ячейке
    onDataLoad: (data) => {}       // Обработчик загрузки данных

    // Параметры для источника данных 'report' (по умолчанию)
    dataSource: 'report',          // Тип источника: 'report' или 'table'
    apiUrl: '/api/endpoint',       // URL API для загрузки данных (для report)

    // Параметры для источника данных 'table'
    dataSource: 'table',           // Тип источника: 'report' или 'table'
    tableTypeId: '335',            // ID типа таблицы (обязательно для dataSource='table')
    parentId: '123',               // ID родителя (опционально, можно передать через GET параметр)

    // Отладка
    debug: false                   // Включить трассировку в консоль (по умолчанию false)
});
```

**ВАЖНО:** Параметр `instanceName` обязателен для корректной работы event handlers (кнопки, пагинация, настройки).

### 3a. Двойная поддержка источников данных

Компонент поддерживает два типа источников данных из Integram API:

#### Вариант 1: Данные из отчета (report) - по умолчанию

```javascript
new IntegramTable('tasks-table', {
    dataSource: 'report',  // можно не указывать, это значение по умолчанию
    apiUrl: '/crm/report/4283?JSON',
    pageSize: 20
});
```

**API endpoint:** `GET /report/{report_id}?JSON&LIMIT={offset},{limit}`

**Формат ответа:**
```json
{
  "columns": [
    { "id": "4284", "type": "3596", "format": "CHARS", "name": "Задача", "granted": 1, "ref": 0 }
  ],
  "data": [
    ["Значение 1"],
    ["Значение 2"]
  ]
}
```

#### Вариант 2: Данные из таблицы (table)

```javascript
new IntegramTable('subordinate-table', {
    dataSource: 'table',
    tableTypeId: '335',     // ID типа таблицы
    parentId: '123',        // ID родительской записи
    pageSize: 20
});
```

**API endpoints:**
- Метаданные: `GET /metadata/{table_type_id}`
- Данные: `GET /object/{table_type_id}/?JSON_DATA&F_U={parent_id}`

**Формат ответа (данные):**
```json
[
  {
    "i": 5300,        // ID записи
    "u": 335,         // ID родителя
    "o": 1,           // Порядковый номер
    "r": [            // Массив значений полей
      "20260202",
      "",
      ""
    ]
  }
]
```

#### Передача Parent ID

Parent ID можно передать тремя способами:

**1. Через параметр конфигурации:**
```javascript
new IntegramTable('my-table', {
    dataSource: 'table',
    tableTypeId: '335',
    parentId: '123'  // Явно указан
});
```

**2. Через GET-параметр URL:**
```
https://example.com/page.html?parentId=456
https://example.com/page.html?F_U=456
https://example.com/page.html?up=456
```

Компонент автоматически считывает значение из URL параметров `parentId`, `F_U` или `up`.

**3. Через data-атрибут:**
```html
<div id="my-table"
     data-integram-table
     data-data-source="table"
     data-table-type-id="335"
     data-parent-id="456"></div>
```

**Приоритет:** Если `parentId` указан в параметрах конфигурации, он имеет приоритет над GET-параметрами URL.

### Data-атрибуты (декларативная инициализация)

При использовании data-атрибутов компонент автоматически инициализируется при загрузке страницы:

| Data-атрибут | Назначение | Пример |
|--------------|------------|--------|
| `data-integram-table` | Маркер для авто-инициализации (обязательно) | - |
| `data-api-url` | URL API для загрузки данных (для dataSource='report') | `"/api/tasks"` |
| `data-data-source` | Тип источника данных: 'report' или 'table' | `"report"` |
| `data-table-type-id` | ID типа таблицы (для dataSource='table') | `"335"` |
| `data-parent-id` | ID родителя (для dataSource='table') | `"123"` |
| `data-page-size` | Количество записей на порции | `"20"` |
| `data-cookie-prefix` | Префикс для cookies | `"my-table"` |
| `data-title` | Заголовок таблицы | `"Задачи"` |
| `data-instance-name` | Имя переменной в window | `"myTable"` |

**Пример 1: Источник данных - отчет (report)**
```html
<div id="tasks"
     data-integram-table
     data-api-url="/crm/report/123?JSON"
     data-page-size="50"
     data-cookie-prefix="tasks-table"
     data-title="Список задач"
     data-instance-name="tasksTable"></div>
```

**Пример 2: Источник данных - таблица (table)**
```html
<div id="subordinate"
     data-integram-table
     data-data-source="table"
     data-table-type-id="335"
     data-parent-id="123"
     data-page-size="20"
     data-cookie-prefix="subordinate-table"
     data-title="Подчиненная таблица"
     data-instance-name="subordinateTable"></div>
```

После загрузки доступ к таблице: `window.tasksTable.loadData()`

**Преимущества:**
- Все параметры в одном месте (в HTML-разметке)
- Не нужно писать отдельный скрипт инициализации
- Удобно для Integram шаблонов с подстановкой `{_global_.z}`

## Автоматическое скрытие колонок

Компонент автоматически скрывает служебные колонки по определенным правилам:

### Колонки с суффиксом "ID"

Если в таблице есть колонка "Клиент" и "КлиентID", колонка "КлиентID" будет автоматически скрыта. Значение ID остается доступным для сохранения изменений, но не отображается пользователю.

### Колонки со стилями

Если в таблице есть колонка "Статус" и "СтатусСтиль" (или "Status" и "StatusStyle"), колонка со стилями будет скрыта, а её значение будет применено как CSS-стиль к соответствующей ячейке в колонке "Статус".

**Пример:**
```
Колонки в API:
- Статус (значение: "Активен")
- СтатусСтиль (значение: "color: green; font-weight: bold;")

Результат:
- Отображается только колонка "Статус"
- Ячейка "Активен" отображается зеленым жирным шрифтом
```

Поиск колонок со стилями **не зависит от регистра** (Style, style, Стиль, стиль - все варианты работают).

## Бесконечный скролл

Компонент использует бесконечный скролл вместо традиционной пагинации:

- **Автоматическая подгрузка** - при прокрутке вниз (за 200px до конца таблицы) автоматически загружаются следующие записи
- **Умное определение наличия данных** - запрашивается `pageSize + 1` записей (например, 21 вместо 20), если пришло 21 - значит есть еще данные
- **Счетчик записей** - внизу отображается "Показано M из N", где:
  - M - количество загруженных записей
  - N - изначально "?" (кликабельный), при клике запрашивается через `RECORD_COUNT=1`
- **Сброс при фильтрации** - при изменении фильтров данные загружаются заново

### Технические детали

Компонент добавляет параметр `LIMIT` к запросу:

- Первая порция: `LIMIT=0,21` (запрашивается 21, отображается 20)
- Вторая порция: `LIMIT=20,21` (если была загружена 21 запись)
- Третья порция: `LIMIT=40,21`

Формат: `LIMIT={offset},{count}`

### Запрос общего количества

Для получения общего количества записей используется параметр `RECORD_COUNT=1`:

```
/api/tasks?RECORD_COUNT=1&FR_4284=test%
```

API возвращает JSON объект с полем `count`:

```json
{
    "count": "142"
}
```

## Фильтрация

Фильтры автоматически применяются при вводе текста с задержкой 500мс (debounce) для оптимизации количества запросов к API.

**Исключение:** Фильтры "Пусто" (`!%`) и "Не пусто" (`%`) применяются **мгновенно** без задержки, при этом текст в поле ввода автоматически очищается.

**Новый дизайн фильтров:** Иконка типа фильтра теперь находится внутри поля ввода слева.

**Кнопка очистки фильтров:**
- Появляется при наличии **любых** активных фильтров, включая "Пусто" и "Не пусто"
- Имеет улучшенный дизайн - круг с символом X
- При клике сбрасывает все фильтры и перезагружает данные

### Типы фильтров

Доступные операторы зависят от типа данных колонки:

#### Для текстовых полей (CHARS, SHORT, MEMO):
- `^` - начинается с... → `FR_{T}={X}%`
- `=` - равно → `FR_{T}={X}`
- `≠` - не равно → `FR_{T}=!{X}`
- `~` - содержит → `FR_{T}=%{X}%`
- `!` - не содержит → `FR_{T}=!%{X}%`
- `!^` - не начинается → `FR_{T}=!%{X}`
- `$` - заканчивается → `FR_{T}=%{X}`
- `(,)` - в списке → `FR_{T}=IN({X})`

#### Для числовых полей (NUMBER, SIGNED):
- `^` - начинается с... → `FR_{T}={X}%`
- `=` - равно
- `≠` - не равно
- `>` - больше
- `<` - меньше
- `≥` - не меньше
- `≤` - не больше
- `...` - в диапазоне → `FR_{T}={X1}&TO_{T}={X2}`

#### Для дат (DATE, DATETIME):
- Аналогично числовым полям
- При выборе операторов `=`, `≥`, `≤`, `>`, `<` поле ввода автоматически заменяется нативным HTML5 date/datetime пикером
- **DATE**: отображается date picker (`input type="date"`) с автоматической конвертацией формата
- **DATETIME**: отображается datetime-local picker (`input type="datetime-local"`) с автоматической конвертацией формата
- При выборе оператора `...` (диапазон) отображается обычное текстовое поле

#### Общие:
- `%` - не пустое → `FR_{T}=%`
- `!%` - пустое → `FR_{T}=!%`

### Пример запроса с фильтрами

```
/api/tasks?LIMIT=0,20&FR_4284=test%&FR_4290=>=100
```

Где:
- `FR_4284=test%` - задачи, начинающиеся с "test"
- `FR_4290=>=100` - ID исполнителя >= 100

## Сортировка

Компонент поддерживает сортировку данных по любому полю таблицы.

### Как использовать

1. **Кликните на название колонки** (не на саму ячейку, а именно на текст заголовка)
2. **Первый клик** - сортировка по возрастанию (▲)
3. **Второй клик** - сортировка по убыванию (▼)
4. **Третий клик** - отключить сортировку

Клик на другую колонку сбрасывает текущую сортировку и включает сортировку по новой колонке.

### Индикация

- ▲ перед названием колонки - сортировка по возрастанию
- ▼ перед названием колонки - сортировка по убыванию

### API параметр

Сортировка передается в запрос GET-параметром `ORDER`:
- По возрастанию: `ORDER={id колонки}`
- По убыванию: `ORDER=-{id колонки}`

```
/api/tasks?LIMIT=0,20&ORDER=4284       // по возрастанию
/api/tasks?LIMIT=0,20&ORDER=-4284      // по убыванию
```

## Группировка

Компонент поддерживает группировку данных по одному или нескольким полям с визуальным объединением ячеек.

### Как использовать

1. **Нажмите кнопку "Группы"** слева от кнопки "Фильтры"
2. **Выберите поля для группировки** в открывшемся окне
3. **Укажите порядок группировки** (первое поле - основная группа, второе - подгруппа и т.д.)
4. **Нажмите "Применить"**

### Особенности режима группировки

- **Отображение**: Группирующие поля отображаются слева объединенными ячейками (rowspan)
- **Лимит данных**: В режиме группировки запрашивается до 1000 записей (бесконечный скролл отключен)
- **Сортировка**: Данные автоматически сортируются по группируемым полям
- **Фильтрация**: По полям группировки можно устанавливать фильтры

### Очистка группировки

Для возврата к обычному виду таблицы нажмите кнопку очистки группировки (✖) рядом с кнопкой "Группы".

## Выбор строк чекбоксами

Компонент поддерживает выделение записей для массовых операций.

### Как использовать

1. **Нажмите иконку чекбоксов** (☑) рядом с иконкой настроек
2. **Слева появится колонка с чекбоксами**
3. **Выберите нужные строки** или используйте чекбокс в заголовке для выбора всех
4. **При выбранных строках появляется кнопка "Удалить"** (красная)

### Массовое удаление

1. Выберите записи чекбоксами
2. Нажмите кнопку "Удалить"
3. Подтвердите удаление
4. Записи удаляются параллельно с отображением прогресса

**Важно**: Если при удалении некоторых записей возникнут ошибки, они будут показаны в диалоге предупреждения в формате "ID записи : значение первой колонки : ответ API".

## Экспорт в Excel/CSV

Компонент поддерживает выгрузку данных таблицы в различные форматы.

### Форматы экспорта

- **XLSX** (Excel) - современный формат Excel
- **XLS** (Excel 97-2003) - для совместимости со старыми версиями
- **CSV** - текстовый формат с разделителями

### Как использовать

1. **Нажмите кнопку экспорта** (📥) в панели управления таблицей
2. **Выберите формат** в выпадающем меню
3. **Файл скачается автоматически**

### Особенности экспорта

- Экспортируются **все данные** таблицы (до 1 000 000 записей), не только отображаемые на экране
- Применяются текущие **фильтры и сортировка**
- Экспортируются только **видимые колонки** в текущем порядке

## Поделиться ссылкой

Компонент позволяет сохранить текущую конфигурацию (фильтры, группировку, сортировку) в URL и поделиться ссылкой.

### Как использовать

1. **Настройте таблицу** - установите фильтры, группировку или сортировку
2. **Появится кнопка копирования** (📋) слева от кнопки "Группы"
3. **Нажмите кнопку** - URL скопируется в буфер обмена
4. **Отправьте ссылку** коллегам - при открытии все настройки восстановятся

### Технические детали

Конфигурация кодируется в base64 и сохраняется в GET-параметре `_itc`:
- Фильтры (тип и значение)
- Колонки группировки (в порядке приоритета)
- Сортировка (колонка и направление)

### Структура конфигурации

```json
{
  "f": {                    // Фильтры
    "colId": {
      "t": "filterType",    // Тип фильтра (^, =, ~, %, и т.д.)
      "v": "filterValue"    // Значение фильтра
    }
  },
  "g": ["col1", "col2"],    // Колонки группировки
  "s": {                    // Сортировка
    "c": "columnId",
    "d": "asc|desc"
  }
}
```

## Редактирование в ячейках

Компонент поддерживает inline-редактирование значений прямо в таблице без открытия модальной формы.

### Как работает

1. **Наведите на редактируемую ячейку** - появится иконка карандаша
2. **Кликните на иконку** - ячейка переключится в режим редактирования
3. **Измените значение** - используйте текстовое поле, выпадающий список или чекбокс в зависимости от типа данных
4. **Сохраните** - нажмите Enter или кликните вне ячейки

### Навигация клавиатурой

- **Tab** - перейти к следующей редактируемой ячейке (с сохранением изменений)
- **Shift+Tab** - перейти к предыдущей редактируемой ячейке (с сохранением изменений)
- **Стрелка вверх** - перейти к ячейке выше в той же колонке
- **Стрелка вниз** - перейти к ячейке ниже в той же колонке
- **Escape** - отменить редактирование

### Типы редакторов

- **Текстовые поля** (SHORT, CHARS) - обычный input
- **Числа** (NUMBER, SIGNED) - input type="text"
- **Даты** (DATE, DATETIME) - HTML5 date/datetime picker
- **Логические** (BOOLEAN) - checkbox
- **Справочники** (ref_id) - выпадающий список с поиском

## Изменение ширины колонок

Компонент поддерживает ручное изменение ширины колонок:

- **Изменение размера**: Наведите на заголовок колонки, появится вертикальная линия справа - потяните её для изменения ширины
- **Минимальная ширина**: 50px
- **Автосохранение**: Ширина колонок автоматически сохраняется в cookies

Ширины колонок сохраняются в том же cookie, что и порядок/видимость колонок.

## Настройки таблицы (⚙️)

Кнопка **⚙️ Настройка** в заголовке таблицы открывает модальное окно с настройками:

### Отступы
- **Просторно** (по умолчанию) - комфортные отступы для чтения
- **Компактно** - уменьшенные отступы для отображения большего количества данных

### Размер страницы
Выберите количество записей, загружаемых за один раз:
- Предустановленные значения: 10, 20, 30, 50, 100
- **Свой вариант** - введите произвольное число

При изменении размера страницы данные перезагружаются автоматически.

### Сокращать длинные значения
- **Да** (по умолчанию) - значения длиннее 127 символов сокращаются с добавлением ссылки "..."
- **Нет** - все значения отображаются полностью

### Скрыть подписи к кнопкам меню
- **Выключено** (по умолчанию) - под иконками кнопок панели управления отображаются текстовые подписи (очистить, группы, фильтры, экспорт, ссылка, вид, колонки)
- **Включено** - подписи скрыты, отображаются только иконки (экономит место в панели)

Настройка позволяет переключаться между компактным (только иконки) и информативным (иконки с подписями) режимами отображения панели управления.

### Сброс настроек
Кнопка **Сбросить настройки** удаляет все сохраненные настройки таблицы и возвращает значения по умолчанию.

Все настройки сохраняются в cookies: `{cookiePrefix}-settings`

## Просмотр полных значений

Когда включено сокращение длинных значений (по умолчанию), текст длиннее 127 символов обрезается с добавлением ссылки "...".

**Просмотр полного текста:**
1. Кликните на ссылку "..." в конце сокращенного значения
2. Откроется модальное окно с полным текстом
3. Закройте окно кликом на кнопку "Закрыть" или на затемненную область

## Липкий заголовок и фильтры

Для удобной работы с длинными таблицами компонент фиксирует заголовок и строку фильтров при вертикальной прокрутке.

**Как работает:**
- **Заголовок таблицы** остается видимым в верхней части при прокрутке вниз
- **Строка фильтров** фиксируется сразу под заголовком
- **Вертикальные разделители** - тонкие светлые линии между колонками для лучшей читаемости
- Используется `position: sticky` с правильными z-index для корректного наложения

Это позволяет всегда видеть названия колонок и иметь доступ к фильтрам без прокрутки наверх.

## Липкий скроллбар

Для широких таблиц, выходящих за пределы экрана, компонент автоматически показывает дополнительный горизонтальный скроллбар внизу окна браузера.

**Как работает:**
- Скроллбар появляется автоматически, когда таблица шире экрана и её нижний скроллбар не виден
- Синхронизируется с основным скроллбаром таблицы
- Исчезает, когда основной скроллбар становится видимым

Это позволяет прокручивать широкие таблицы без необходимости скроллить страницу вниз.

## Сохранение состояния

Компонент автоматически сохраняет в cookies:
- Порядок колонок
- Видимость колонок
- Ширины колонок
- Режим отступов (компактный/просторный)
- Размер страницы
- Режим сокращения длинных значений
- Скрытие подписей кнопок меню
- Видимость полей в формах редактирования (для каждого типа)

Cookie имеют формат:
- `{cookiePrefix}-state` - порядок, видимость и ширины колонок
- `{cookiePrefix}-settings` - настройки таблицы
- `{cookiePrefix}-form-fields-{typeId}` - видимость полей формы для типа

Срок хранения: 1 год

## Методы API

После создания экземпляра доступны следующие методы:

```javascript
const table = new IntegramTable('my-table', {...});

// Перезагрузить данные (сброс до начала)
table.loadData();

// Загрузить следующую порцию (append mode)
table.loadData(true);

// Запросить общее количество записей
table.fetchTotalCount();

// Включить/выключить фильтры
table.toggleFilters();

// Проверить наличие активных фильтров
table.hasActiveFilters(); // возвращает true/false

// Очистить все фильтры
table.clearAllFilters();

// Открыть настройки колонок
table.openColumnSettings();

// Закрыть настройки колонок
table.closeColumnSettings();

// Открыть настройки таблицы
table.openTableSettings();

// Закрыть настройки таблицы
table.closeTableSettings();

// Сбросить настройки таблицы к значениям по умолчанию
table.resetSettings();

// Показать полное значение сокращенного текста
table.showFullValue(event, fullValue);

// Открыть форму редактирования
table.openEditForm(recordId, typeId, rowIndex);

// Открыть настройки видимости полей формы
table.openFormFieldSettings(typeId, metadata);

// Сохранить состояние колонок в cookies
table.saveColumnState();

// Загрузить состояние колонок из cookies
table.loadColumnState();

// Сохранить настройки таблицы в cookies
table.saveSettings();

// Загрузить настройки таблицы из cookies
table.loadSettings();

// --- Сортировка ---

// Переключить сортировку колонки (asc -> desc -> none)
table.toggleSort(columnId);

// --- Группировка ---

// Открыть настройки группировки
table.openGroupingSettings();

// Очистить группировку
table.clearGrouping();

// Проверить наличие активных фильтров или групп
table.hasActiveFiltersOrGroups(); // возвращает true/false

// --- Поделиться ссылкой ---

// Получить URL с текущей конфигурацией
table.getConfigUrl();

// Скопировать URL с конфигурацией в буфер обмена
table.copyConfigUrl();

// --- Выбор строк ---

// Включить/выключить режим чекбоксов
table.toggleCheckboxMode();

// Удалить выбранные записи
table.deleteSelectedRecords();

// --- Экспорт ---

// Показать/скрыть меню экспорта
table.toggleExportMenu(event);

// Экспортировать таблицу в указанном формате
table.exportTable('xlsx'); // 'xlsx', 'xls', 'csv'

// Сохранить настройки видимости полей формы
table.saveFormFieldVisibility(typeId, visibility);

// Загрузить настройки видимости полей формы
table.loadFormFieldVisibility(typeId);
```

## Решение проблем

### Данные не загружаются

1. Проверьте формат ответа API - он должен содержать `columns`, `data` и `total`
2. Откройте Developer Tools → Network и проверьте запрос
3. Убедитесь, что API поддерживает параметр `LIMIT`
4. Для получения общего количества API должен поддерживать `RECORD_COUNT=1` и возвращать простое число

### Бесконечный скролл не работает

1. Проверьте, что API возвращает `pageSize + 1` записей при наличии данных
2. Убедитесь, что не заблокированы события `scroll` на window
3. Проверьте в консоли браузера наличие ошибок JavaScript

### Колонки не перетаскиваются

1. Проверьте, что браузер поддерживает HTML5 Drag and Drop API
2. Убедитесь, что не заблокированы события dragstart/dragover/drop

### Фильтры не работают

1. Проверьте, что API корректно обрабатывает параметры `FR_{columnId}`
2. Для диапазонов также должен поддерживаться параметр `TO_{columnId}`

### Настройки не сохраняются

1. Проверьте, что cookies разрешены в браузере
2. Убедитесь, что `cookiePrefix` уникален для каждой таблицы на странице

### Кнопка "Сбросить настройки" не работает

**Исправлено в последней версии:**
- Кнопка теперь корректно удаляет cookie с состоянием колонок (`{cookiePrefix}-state`)
- После сброса состояние колонок возвращается к исходному

### Кнопка "Очистить фильтры" не работает

**Исправлено в последней версии:**
- Кнопка теперь корректно сбрасывает все фильтры и перезагружает данные
- Работает для всех типов фильтров, включая "Пусто" и "Не пусто"
- Правильно очищает внутреннее состояние и обновляет UI

### Колонки со стилями появляются после перезагрузки

**Исправлено в последней версии:**
- Колонки с суффиксом "Стиль" теперь остаются скрытыми при загрузке состояния из cookies
- Стили продолжают применяться к соответствующим ячейкам

### Специальные символы в полном тексте вызывают ошибку

**Исправлено в последней версии:**
- JavaScript специальные символы (`'`, `"`, `\`, переводы строк) корректно экранируются
- Функция `showFullValue` теперь безопасно обрабатывает любой текст

## Редактирование записей

Компонент поддерживает модальные формы для редактирования существующих записей. Для колонок, у которых есть одноименная колонка с суффиксом "ID", автоматически появляется возможность редактирования.

### Как это работает

1. **Определение редактируемых колонок**: Если в таблице есть колонка "Клиент" и "КлиентID", то колонка "Клиент" становится редактируемой
2. **Иконка редактирования**: При наведении на ячейку с существующей записью (ID не пустой) в правом нижнем углу появляется иконка редактирования (карандаш)
3. **Открытие формы**: Кликните на иконку редактирования для открытия формы редактирования записи

**Примечание**: Создание новых записей временно отключено - иконка редактирования не отображается для ячеек с пустым ID.

### Типы полей

Компонент автоматически определяет тип поля по символьному значению Format (из таблицы "Форматы колонок") и отображает соответствующий элемент управления:

- **Текстовые поля** (SHORT, CHARS) - обычный input
- **Многострочные поля** (MEMO) - textarea (4 строки)
- **Числовые поля** (NUMBER, SIGNED) - input type="text"
- **Дата** (DATE) - HTML5 date picker (календарь) с форматом ДД.ММ.ГГГГ
- **Дата и время** (DATETIME) - HTML5 datetime-local picker (календарь с временем) в формате ДД.ММ.ГГГГ ЧЧ:ММ (с округлением до 5 минут)
- **Логические поля** (BOOLEAN) - checkbox
- **Справочники** (ref_id) - выпадающий список с данными из связанной таблицы
- **Подчиненные таблицы** (arr_id) - иконка таблицы с количеством записей (только для просмотра)

**Определение типа поля:**
- API может возвращать как числовой ID типа (например, `12` для MEMO), так и символьное название (`"MEMO"`)
- Компонент автоматически обрабатывает оба варианта:
  - Если приходит символьное название - использует его напрямую
  - Если приходит числовой ID - преобразует в символьное название по таблице "Форматы колонок"
- Символьные названия типов не преобразуются и используются как есть

**Работа с календарями (Date/DateTime Pickers):**
- Поля DATE и DATETIME используют нативные HTML5 виджеты выбора даты/времени
- **DATE**: отображается календарь с выбором даты (input type="date")
- **DATETIME**: отображается календарь с выбором даты и времени (input type="datetime-local")
- Время в DATETIME округляется до 5 минут (step="300")
- **Автозаполнение текущей датой/временем:**
  - При создании новой записи поля DATE и DATETIME **только в первой колонке** автоматически заполняются текущим значением
  - Остальные поля DATE/DATETIME остаются пустыми (можно задать defaults на сервере)
  - При редактировании используются существующие значения
- Автоматическая конвертация форматов:
  - Из формата API (различные варианты) в HTML5 формат (YYYY-MM-DD или YYYY-MM-DDTHH:MM)
  - Из HTML5 формата обратно в формат API (ДД.ММ.ГГГГ или ДД.ММ.ГГГГ ЧЧ:ММ)
- Скрытое поле хранит значение в формате API для отправки на сервер
- **Консистентный формат в таблицах**: DATE отображается как dd.mm.yyyy, DATETIME как dd.mm.yyyy hh:mm:ss

**Пропуск пустых параметров:**
- При создании новой записи пустые поля **не передаются** на сервер
- Это позволяет серверу применить значения по умолчанию (defaults) из настроек БД
- Передаются только поля с заполненными значениями

### API endpoints

Компонент использует следующие API endpoints:

- `GET /metadata/{typeId}` - получение метаданных о типе (структура таблицы)
- `GET /edit_obj/{recordId}?JSON` - получение данных записи для редактирования
- `GET /_ref_reqs/{requisiteId}` - получение опций для справочников
- `POST /_m_new/{typeId}?JSON&up={parentId}` - создание новой записи
- `POST /_m_save/{recordId}?JSON` - сохранение изменений записи
- `POST /_m_del/{recordId}?JSON` - удаление записи

### Поиск в справочниках

Для полей-справочников (ref_id) в форме редактирования доступен живой поиск:

1. **Начните вводить текст** в поле справочника
2. **Список автоматически фильтруется** по введенному тексту
3. **Выберите нужный вариант** из отфильтрованного списка
4. **Поиск регистронезависимый** и ищет по любой части названия

Это упрощает работу со справочниками, содержащими большое количество элементов.

### Настройка видимости полей

В форме редактирования доступна кнопка настройки полей (⚙️ в нижнем левом углу):

1. **Откройте настройки**: Кликните на кнопку ⚙️ в футере формы
2. **Выберите поля**: Отметьте галочками поля, которые должны отображаться
3. **Сохраните**: Кликните "Сохранить"
4. **Форма обновится**: Скрытые поля больше не отображаются

**Особенности:**
- Настройки сохраняются отдельно для каждого типа записей
- Скрытые поля не участвуют в валидации
- Настройки хранятся в cookies и применяются автоматически при следующем открытии формы

### Порядок кнопок

В формах редактирования кнопки расположены в следующем порядке:
- **Сохранить** (основная кнопка, слева)
- **Отменить** (вторичная кнопка, справа)

### Удаление записи

В форме редактирования можно удалить запись:

1. **Включите отображение кнопки**: В настройках формы (⚙️) установите галочку "Показывать кнопку Удалить"
2. **Кнопка "Удалить"** появится в форме редактирования (красная)
3. **Подтвердите удаление** в диалоговом окне
4. **Запись удалится** через API `POST /_m_del/{recordId}?JSON`

**Важно**: По умолчанию кнопка удаления скрыта для безопасности.

### Автоматическое обновление

После успешного сохранения компонент автоматически обновляет данные таблицы, отображая актуальную информацию.

## Подчиненные таблицы

Компонент поддерживает отображение и редактирование подчиненных (связанных) таблиц прямо в форме редактирования записи через систему вкладок.

### Как это работает

При открытии формы редактирования записи компонент автоматически определяет наличие подчиненных таблиц по полю `arr_id` в метаданных (запрос `/metadata/{typeId}`).

**Структура вкладок:**
1. **Первая вкладка "Атрибуты"** - содержит обычные поля записи (текстовые, числовые, справочники и т.д.)
2. **Вкладки подчиненных таблиц** - каждая подчиненная таблица отображается на отдельной вкладке с названием и количеством записей в скобках (например, "Задачи (5)")

### Получение данных

Для каждой подчиненной таблицы компонент выполняет следующие запросы:

1. **Метаданные таблицы:**
   ```
   GET /metadata/{typeId}
   ```
   Возвращает структуру колонок подчиненной таблицы

2. **Данные таблицы:**
   ```
   GET /object/{typeId}/?JSON_DATA&F_U={parentId}
   ```
   Где `parentId` - ID родительской записи. Возвращает массив записей:
   ```json
   [
     {
       "i": 5151,           // ID записи
       "u": 333,            // ID родителя
       "o": 1,              // порядковый номер
       "r": ["val1", "val2"] // массив значений полей
     }
   ]
   ```

### Отображение подчиненных таблиц

- **Вложенные подчиненные таблицы** - если у подчиненной таблицы есть свои подчиненные таблицы, они отображаются только как число записей в скобках "(N)" без возможности провалиться глубже
- **Редактирование записей** - клик на первую колонку строки открывает форму редактирования этой записи
- **Кнопка "Добавить"** - позволяет создать новую запись в подчиненной таблице (автоматически передается `up={parentId}`)
- **Удаление записей** - доступно через форму редактирования записи

### Многоуровневые модальные окна

Компонент корректно обрабатывает ситуации, когда открывается несколько модальных окон друг над другом:

- **Правильное наложение** - каждое новое окно отображается поверх предыдущего с корректным z-index
- **Независимые overlay** - каждое окно имеет свой затемняющий фон
- **Каскадное закрытие** - закрытие верхнего окна возвращает фокус к предыдущему

### Автоматическое расширение модальных окон

Если подчиненная таблица не помещается в стандартное модальное окно (требуется вертикальная прокрутка), окно автоматически расширяется:

- **Обычная ширина** - 600px для простых форм
- **Расширенная ширина** - до 93% ширины экрана для форм с подчиненными таблицами
- **Адаптивная высота** - модальное окно подстраивается под содержимое

Это обеспечивает комфортную работу с большими таблицами без необходимости прокрутки внутри модального окна.

### Защита от CSRF

- Можно закрыть кликом на уведомление

## Глобальная функция создания записей

Компонент предоставляет глобальную функцию `openCreateRecordForm()` для открытия формы создания записи из любого места на странице, без необходимости наличия таблицы.

### Использование

```javascript
// Базовое использование - открыть форму для таблицы 3596 с родителем 1
openCreateRecordForm(3596, 1);

// С предзаполненными значениями полей
openCreateRecordForm(3596, 1, {
    't3888': 357,           // Справочное поле (ID значения)
    't3886': 'Значение'     // Текстовое поле
});
```

### Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `tableTypeId` | number | Да | ID типа таблицы, в которой создается запись |
| `parentId` | number | Да | ID родительской записи (передается в параметр `up`) |
| `fieldValues` | object | Нет | Объект с предзаполненными значениями полей в формате `{'t{fieldId}': value}` |

### Особенности

- Работает без наличия таблицы на странице
- Поддерживает все типы полей: текст, справочники, boolean, дата, время, memo, файлы
- После успешного создания автоматически обновляет все таблицы IntegramTable на странице
- Автоматически загружает метаданные типа для построения формы
- Поддерживает отображение информации о родительской записи в заголовке формы

## Редактирование колонок (форма редактирования колонки)

Компонент поддерживает полное управление структурой таблицы через форму редактирования колонки. Доступ к форме — через значок ✏️ в строке настроек колонок.

### Доступные операции

- **Переименование колонки** — изменить отображаемое имя колонки (доступно для всех колонок, включая первую)
- **Изменение базового типа** — выбрать тип данных из списка доступных типов
- **Уникальные значения** — установить флаг уникальности (колонка будет принимать только уникальные значения)
- **Обязательно к заполнению** — установить флаг обязательности (для первой колонки всегда включено)
- **Псевдоним** (только для ссылочных колонок) — задать псевдоним поля
- **Мультиселект** (только для ссылочных колонок) — разрешить выбор нескольких значений

### API endpoints

- `POST /_d_save/{columnId}?JSON` — сохранение имени колонки (`val` = новое имя)
- `POST /_d_save/{columnId}?JSON` — сохранение типа колонки (`typ` = ID типа, `unique` = 0/1)
- `POST /_d_save/{columnId}?JSON` — установка обязательности (`required` = 0/1)
- `POST /_d_save/{columnId}?JSON` — установка псевдонима (`alias` = строка)
- `POST /_d_save/{columnId}?JSON` — переключение мультиселекта (`multi` = 0/1)
- `POST /_d_del/{columnId}?JSON` — удаление колонки
- `POST /_d_del/{columnId}?JSON&force=1` — удаление колонки вместе с данными

## Хуки и переопределения (integramTableOverrides)

Компонент поддерживает кастомизацию поведения через глобальный объект `window.integramTableOverrides`.

### Переопределение URL справочников (ddls)

Позволяет указать альтернативный URL для загрузки опций справочных полей:

```javascript
window.integramTableOverrides = {
    ddls: {
        // Для реквизита с ID 3888 использовать альтернативный URL
        '3888': '/api/custom/statuses',
        // Поддержка параметризации
        '3889': '/api/custom/users?filter={value}'
    }
};
```

### Хуки полей (fieldHooks)

Позволяют динамически фильтровать справочники в зависимости от значений других полей:

```javascript
window.integramTableOverrides = {
    fieldHooks: {
        // При изменении поля с ID 3888 обновить фильтрацию поля 3889
        '3888': function(value, form) {
            // Вернуть дополнительные параметры для запроса справочника 3889
            return {
                '3889': { filter: value }
            };
        }
    }
};
```

### Пример использования

```html
<script>
    // Настроить переопределения перед инициализацией таблицы
    window.integramTableOverrides = {
        ddls: {
            '3888': '/crm/custom/statuses'
        },
        fieldHooks: {
            '3888': function(statusValue, formElement) {
                // Фильтровать исполнителей по статусу
                return {
                    '3889': { status: statusValue }
                };
            }
        }
    };
</script>
<script src="/js/integram-table.js"></script>
```
