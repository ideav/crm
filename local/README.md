# Локальная установка Интеграм — начните отсюда

Этот комплект разворачивает Интеграм на сервере клиента. Есть два пути — выберите один.

| Путь | Кому | Что нужно на сервере | Инструкция |
|------|------|----------------------|------------|
| **1. Docker (рекомендуется)** | Если на сервере есть или можно поставить Docker | Docker Engine + Docker Compose, свободные порты 8080/8443 | [УСТАНОВКА-DOCKER.md](УСТАНОВКА-DOCKER.md) |
| **2. Без Docker (вручную)** | Если Docker ставить нельзя, есть обычный LAMP | Ubuntu/Debian, Apache + PHP 8.2 + MariaDB/MySQL | [УСТАНОВКА-БЕЗ-DOCKER.md](УСТАНОВКА-БЕЗ-DOCKER.md) |

Оба пути дают одинаковый результат: рабочий Интеграм на этом сервере. Первый вход —
открыть адрес сервера → на странице входа нажать «Войти» → логин `admin`, пароль —
тот, который специалист задаст при установке (попадёте в рабочую область `my`).

## Что в архиве

```
index.php              — приложение (монолит)
include/               — подключение к БД и хелперы (конфиг читается из окружения)
js/ css/ i/ assets/    — фронтенд (скрипты, стили, иконки, сторонние библиотеки)
ace/                   — редакторы файлов диспетчера (dir_admin): текстовый ACE (работает офлайн) и
                         визуальный GrapesJS (тянет библиотеку из интернета/CDN — нужен доступ в сеть)
crm/ templates/        — HTML-шаблоны рабочих мест
download/              — папка пользовательских файлов (пустая, заполняется при работе)
Dockerfile             — образ приложения (PHP 8.2 + Apache)
compose.yaml           — сервисы app + БД (MariaDB) для Docker-пути
docker/                — конфиг Apache, инициализация БД, entrypoint
  ├─ apache/integram.conf
  ├─ mysql/010-integram-bootstrap.sql   ← начальная схема БД (нужна и для ручной установки)
  └─ entrypoint.sh
scripts/install-local-docker.sh         — установщик в один шаг (Docker-путь)
.env.docker.example    — шаблон настроек
local/                 — эти инструкции и вспомогательные файлы
  ├─ README.md                 (этот файл)
  ├─ УСТАНОВКА-DOCKER.md
  ├─ УСТАНОВКА-БЕЗ-DOCKER.md
  ├─ make-archive.sh           (для нас: как собран этот архив)
  └─ files/                    (шаблоны для ручной установки)
```

## Безопасность (важно перед установкой)

- **Master-пароль и SALT** задаёт специалист при установке; в присланных файлах их нет.
  Не используйте примеры из `.env.docker.example` на рабочей установке.
- По умолчанию HTTPS работает с **самоподписанным сертификатом** — браузер покажет
  предупреждение. Для рабочей эксплуатации замените сертификат на сертификат клиента
  (см. раздел «Сертификат» в инструкциях).
- Порты `8080` (HTTP) и `8443` (HTTPS) — при необходимости поменяйте на 80/443 или
  закройте HTTP-порт после проверки.

## Кого звать при проблемах

Установку выполняет системный администратор клиента по инструкции. Если что-то не
сходится — сверьте вывод команды с разделом «Проверка» соответствующей инструкции и
пришлите нам текст ошибки (не скриншот) и вывод указанных в инструкции команд.
